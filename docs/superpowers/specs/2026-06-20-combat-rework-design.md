# 战斗重构设计：三拍拆招、成功产彩与终结分镜切入

日期：2026-06-20  
状态：等待用户复审  
原始记录：[[inspiration/2026-06-20_combat_identity_and_animation_brainstorm_qa|战斗特色化与动画爽感头脑风暴 Q&A]]

## 目的

本设计用于重构《剑与黑塔》的当前卡牌战斗方向，使战斗不再主要依赖传统“玩家回合连续出牌、魔物回合统一结算意图”的牌组构筑框架。新的战斗核心卖点为：

> 三拍拆招 + 成功产彩 + 终结分镜切入。

第一版实现切片刻意 **不包含彩能附魔与五色终结转译**。本阶段先验证新的战斗节奏：读取魔物动作、用玩家拍位截住魔物动作、从成功动作中生成彩能，并在战斗过程中手动切入终结牌。

## 设计目标

- 区别于《杀戮尖塔》：用基于拍位的动作编排，替代长玩家回合和行动点出牌序列。
- 区别于《废墟图书馆》：动作对撞本身不是唯一奖励；核心奖励是“成功动作生成彩能，彩能驱动手动终结切入”。
- 让战斗像一页由玩家卡牌选择驱动的黑白漫画：格挡、闪避、反击、打出空门，再切入终结分镜。
- 让第一版 MVP 足够小，先验证可读性和爽感，再恢复彩能附魔。

## 核心循环

每轮战斗分为四个阶段：

1. **预告阶段**  
   魔物公开本轮动作序列。普通魔物默认完整公开动作。精英和 Boss 未来可以通过明确技能或被动隐藏特定拍位。

2. **编排阶段**  
   玩家把行动牌放入拍位。初始武器左轮剑与机械臂均提供 3 拍。每拍放 1 张行动牌。允许空拍。行动牌在空拍位上松开后，立即以该拍位作为临时锚点拉出目标箭头；右键或点击空白取消时，该行动牌退回手牌。玩家还可以把 1 张终结牌放入终结槽。

3. **结算阶段**  
   每个已放行动牌的玩家拍位需要在编排中选择一个未重复锁定的魔物拍位作为对撞目标。被锁定的玩家动作与魔物动作对撞。未被锁定的魔物拍仍按魔物自身原本拍序结算。若某个魔物的全部拍位都已被锁定，剩余玩家拍位可以直接攻击该魔物本体。

4. **轮末阶段**  
   已结算的行动牌进入弃牌堆。终结槽中的终结牌默认在轮末弃掉。彩能默认在轮末清空。特殊效果可以保留终结槽或部分彩能。

## 卡牌与动作

玩家可见卡牌类型继续保留两类：

- **行动牌**：放入拍位。
- **终结牌**：放入终结槽。

行动牌是一个或多个动作的容器。一拍只能放 1 张牌，但这张牌内部可以包含多个按顺序结算的动作。

攻击动作：

- 斩击
- 钝击
- 弹射

非攻击动作：

- 格挡
- 闪避

示例：

- `格挡 4 -> 钝击 6`
- `闪避 50% -> 斩击 3 x2`

魔物拍位也使用卡牌化动作块表现，但魔物不使用卡组、手牌、抽牌堆或弃牌堆。魔物动作卡由遭遇数据和 AI 行为生成。

## 对撞规则

当玩家拍位锁定一个魔物拍位时，双方动作序列按顺序结算：

- 攻击 vs 攻击：双方各自造成对应攻击伤害，并播放拼刀 / 对攻演出。
- 攻击 vs 格挡：攻击数值减去格挡数值，剩余部分造成伤害。魔物格挡玩家攻击时同理。
- 攻击 vs 闪避：闪避基础有 50% 概率完全避开伤害。武器、遗物、卡牌或状态可以修正该概率。
- 格挡 / 闪避 vs 格挡 / 闪避：双方动作互相消耗，不造成伤害。
- 一方动作序列先耗尽时，另一方剩余动作直接生效。

当玩家拍位直接攻击魔物本体时，只结算该行动牌内的攻击动作。该牌内的格挡和闪避动作会被跳过。

## 魔物抗性

每个魔物对三种攻击类型拥有三档抗性：

- **抵抗**：造成 50% 伤害。
- **标准**：造成 100% 伤害。
- **弱点**：造成 150% 伤害。

斩击、钝击、弹射伤害都使用这套抗性表。终结牌的攻击伤害也受攻击类型抗性影响。

魔物抗性通过悬停魔物查看。玩家拖拽或瞄准行动牌 / 终结牌时，UI 应临时提示当前攻击类型对目标是抵抗、标准还是弱点。

## 彩能

第一版战斗重构 MVP 中，所有彩能都视为无色彩能。彩能附魔和五色转译延后实现。

彩能只由成功动作生成：

- 攻击动作成功造成伤害。
- 格挡动作有效降低受到的伤害。
- 闪避动作成功躲开攻击。

每个成功动作固定生成 1 点无色彩能。基础彩能上限保持 6。若彩能已满，后续成功动作可能浪费产能，除非某个效果扩展上限或通过终结牌消耗彩能。

彩能默认在轮末清空。保留彩能是高价值能力，只能由明确的卡牌、遗物、状态、武器效果或终结槽规则提供。

## 终结槽

终结牌不再作为普通拍位牌打出。

- 编排阶段，玩家可以把 1 张终结牌放入终结槽。
- 终结牌不占用玩家拍位。
- 每次玩家动作成功后，若当前彩能满足终结牌条件，终结槽进入可释放状态。
- 玩家可以手动点击终结牌，切入漫画分镜演出。
- 释放终结牌会消耗该终结牌要求的彩能。
- 同一张终结牌在一轮内可以多次释放，只要玩家反复重新生成足够彩能，再次满足释放条件。
- 释放终结牌不会消耗终结牌本身。
- 终结槽中的终结牌默认在轮末弃掉。
- 特殊效果可以让终结槽跨轮保留。

UI 不应使用强制弹窗打断。可释放终结牌应通过高亮、脉冲、短暂停顿或轻微慢动作提示玩家，而不是强制暂停等待确认。

## 手牌与牌库流转

- 战斗开始时抽 5 张牌。
- 手牌默认跨轮保留。
- 每轮开始时抽 2 张牌。
- 已完成结算的行动牌进入弃牌堆。
- 允许空拍。
- 抽牌堆不足时，默认可将弃牌堆洗回抽牌堆继续抽，除非后续规则另有说明。

第一版战斗重构 MVP 不使用行动点。拍位数量、手牌可用性、卡牌动作内容、魔物压力和终结牌彩能条件是主要限制。

## 多魔物目标规则

每个玩家拍位选择一个目标：

- 任一魔物的一个未重复锁定拍位。
- 或者，在该魔物全部拍位都已被锁定后，直接选择该魔物本体。

基础规则：

- 每个魔物拍位最多只能被一个玩家拍位锁定。
- 玩家可以选择任意魔物的任意拍位，不限于相同序号的拍。
- 未被锁定的魔物拍位仍按魔物自身原本拍序结算。
- 若某个魔物的全部拍位都被锁定，剩余玩家拍位可以直接攻击该魔物本体。

这形成核心战术模式：先拆危险招式，再攻击空门。

## 魔物动作卡作为奖励

魔物动作卡不是魔物卡组的一部分，但它们是内容资产。后续击败魔物时，部分魔物动作卡可以作为玩家奖励。

如果魔物卡进入玩家牌组：

- 它成为行动牌。
- 它可以放入玩家拍位。
- 它遵循同样的成功动作产彩规则。
- 当彩能附魔系统回归后，它也可以接受色彩附魔。

第一版战斗重构 MVP 应为卡牌来源预留数据字段，但不需要实现魔物卡奖励。

## UI 与动画

战斗 UI 应呈现为三拍漫画分镜：

- 玩家侧显示 3 个拍位。
- 魔物侧每个魔物显示本轮动作序列。
- 玩家把行动牌拖入拍位并松开后，系统立即从该拍位拉出箭头，玩家选择魔物拍位或已露出空门的魔物本体作为目标；右键或点击空白取消并让该行动牌退回手牌。
- 被锁定的魔物拍显示“已接招”状态。
- 未锁定魔物拍显示危险提示。
- 全部拍位被锁定的魔物显示“空门”状态。
- 行动牌显示内部动作序列。
- 6 格彩能槽仍常驻显示。
- 终结槽位于彩能槽附近。

动画原则：

- 普通拍位演出应表现格挡、闪避、对攻、反击和追击。
- 成功动作应把彩能反馈送入彩能槽。
- 终结牌切入更大的漫画分镜，使用速度线、局部特写、命中停顿和清晰的伤害 / 彩能反馈。
- 动画不能遮挡尚未结算的魔物拍、生命、彩能或终结槽。

## 第一版战斗重构 MVP 范围

包含：

- 玩家角色：左。
- 武器：左轮剑与机械臂。
- 两把初始武器都使用 3 拍。
- 不使用行动点。
- 战斗开始抽 5。
- 每轮开始抽 2。
- 手牌保留。
- 只保留行动牌和终结牌。
- 动作类型：斩击、钝击、弹射、格挡、闪避。
- 魔物抗性：抵抗 / 标准 / 弱点。
- 无色彩能上限 6。
- 成功动作固定生成 1 点无色彩能。
- 1 个终结槽。
- 成功动作后，若满足彩能条件，可释放终结牌。
- 普通魔物完整公开动作序列。
- 多魔物支持可以先限制到最多 2 个魔物。

不包含：

- 色彩碎片。
- 行动牌色彩附魔。
- 五色终结转译。
- 隐藏魔物拍位。
- 魔物卡奖励。
- 完整 6 战 Run 替换。
- 多个终结槽。
- 行动点或替代费用资源。

推荐验证切片：

- 1 场教学战：教学拍位、目标选择、格挡、攻击和彩能生成。
- 1 场双魔物战：教学拆招目标选择和空门攻击。
- 1 场精英 / Boss 概念战：教学终结时机和抗性目标选择。

## 数据模型影响

行动牌需要结构化动作列表：

- 动作类型：斩击、钝击、弹射、格挡、闪避。
- 数值。
- 命中次数或重复次数。
- 目标规则。
- 可选成功触发效果。

魔物需要每轮或每个行为状态的动作序列：

- 拍位序号。
- 魔物动作卡 ID。
- 公开状态。
- 可选特殊规则。

终结牌需要：

- 彩能需求。
- 若造成攻击伤害，则需要攻击类型。
- 效果负载。
- 是否有特殊释放时机限制。
- 终结槽是否由该牌或其他效果保留。

第一版 MVP 可以保留颜色字段但不启用。后续彩能附魔回归时，颜色应绑定整张行动牌，而不是绑定单个动作。

## 风险

- **UI 过载**：拍位、魔物动作卡、目标连线、动作序列、抗性、彩能和终结槽会竞争注意力。
- **平衡爆发**：若产彩过于容易，一轮内无限次释放终结牌可能压过普通动作。
- **闪避波动**：50% 闪避如果没有清晰概率和反馈，可能影响失败复盘。
- **实现范围**：这是战斗循环重构，不是现有卡牌出牌系统的小迭代。
- **竞品联想**：动作对撞容易让人联想到《废墟图书馆》。设计必须突出任意拍拆招、成功产彩和手动终结切入。

## 后续未决问题

- “拍位”“拆招”“空门”“终结槽”“成功动作”等中英文 UI 术语的最终命名。
- 魔物是否可以拥有不同于玩家的拍数。
- 精英 / Boss 隐藏拍位未来用问号、剪影还是假预告表现。
- 彩能附魔如何在第一版 MVP 后回归。
- 魔物卡奖励进入普通奖励、稀有奖励，还是独立战利品轨道。
- 是否允许个别终结牌作为单卡例外设置每轮释放次数上限。
