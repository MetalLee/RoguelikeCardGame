# 三拍槽位与魔物拍瞄准 UI 设计

日期：2026-06-21
状态：已由开发者确认
关联设计：[[docs/superpowers/specs/2026-06-20-combat-rework-design|战斗重构设计：三拍拆招、成功产彩与终结分镜切入]]

## 背景

当前 `combat-rework-beat` 分支已经接入三拍拆招的规则核心与最小战斗 UI，但战斗场景仍有两个明显缺口：

- 手牌不能拖入玩家拍位，导致玩家无法真正编排三拍。
- 魔物本轮动作拍位没有在战斗场景中显示，导致玩家无法读招、选招和确认对撞目标。

本设计补齐第一版可玩交互闭环，目标是让玩家能完成“手牌填拍 -> 自动拍位瞄准 -> 魔物拍对撞 -> 结束结算”的最小流程。后续交互细节由 [[docs/superpowers/specs/2026-06-21-beat-placement-auto-target-design|拍位放入后自动瞄准交互设计]] 修订：行动牌放入空玩家拍位后不需要再次点击拍位，系统立即从该拍位拉出目标箭头。

## 确认交互

采用“两步式连续操作”：

1. 玩家先将手牌中的行动牌拖入一个空的玩家拍位。
2. 在该拍位上松开鼠标后，规则层先放入行动牌。
3. 该玩家拍位立即进入临时瞄准状态，显示一条从该玩家拍位出发并跟随鼠标的箭头。
4. 玩家点击一个合法目标完成瞄准。
5. 若玩家右键或点击空白区域，本次刚放入的行动牌退回手牌，该玩家拍位清空。

合法目标分两类：

- 若目标魔物仍有未锁定魔物拍，玩家只能选择该魔物的一个未锁定拍作为对撞目标。
- 若目标魔物所有魔物拍都已被锁定，玩家可以选择该魔物本体作为目标。

即使魔物所有拍都已被锁定，玩家也必须先把行动牌放入空的玩家拍位，再通过该拍位拉箭头选择魔物本体。手牌不允许直接打魔物本体。

## UI 表达

玩家侧：

- 三个玩家拍位常驻显示。
- 空拍显示“第 N 拍”。
- 放入行动牌后显示卡名或短卡牌 ID。
- 临时瞄准中的玩家拍位高亮。
- 已选择目标的玩家拍位显示简短目标提示，例如“接：魔物拍 1”或“攻：本体”。

魔物侧：

- 每个存活魔物显示本轮魔物拍位。
- 魔物拍位显示动作串，例如“斩击 4”或“格挡 3 -> 钝击 5”。
- 已被锁定的魔物拍显示“已接招”状态，并不允许再次选择。
- 可被当前玩家拍选择的魔物拍高亮。
- 当某个魔物所有拍都已锁定时，魔物本体进入可选“空门”状态。

箭头：

- 只在玩家拍位待选择目标时显示。
- 起点为玩家拍位中心或上边缘。
- 终点跟随鼠标。
- 悬停合法魔物拍或魔物本体时，目标高亮。
- 点击合法目标后箭头消失；右键或点击空白取消时箭头也消失。

## 数据与状态

UI 更新 `CombatState.BeatRound.PlayerBeats`：

- 拖入行动牌时，更新对应 `PlayerBeatSlot.CardInstanceId` 和 `CardId`。
- 选择魔物拍时，写入 `BeatTarget { Kind = EnemyBeat, EnemyInstanceId, EnemyBeatIndex }`。
- 选择魔物本体时，写入 `BeatTarget { Kind = EnemyBody, EnemyInstanceId }`。
- 取消临时瞄准时，撤销刚放入的 `PlayerBeatSlot.CardInstanceId` 和 `CardId`，并把该 `CardInstanceId` 放回手牌。

目标合法性继续依赖 `BeatCombatService.ValidatePlayerBeatTargets`。UI 只负责避免明显非法选择，规则服务负责最终校验。

## 暂不实现

本次只做第一版可玩闭环，不实现以下内容：

- 从手牌直接拖到魔物拍。
- 直接从手牌攻击魔物本体。
- 终结槽拖拽、点击释放或漫画分镜完整演出。
- 多种箭头动画、复杂吸附或高级拖拽手感。
- 隐藏魔物拍、Boss 特殊遮蔽拍。

## 验收标准

- 战斗首屏可看到玩家拍位和魔物拍位。
- 行动牌可拖入空玩家拍位。
- 放入拍位后出现跟随鼠标的目标箭头。
- 右键或点击空白区域取消临时瞄准后，行动牌退回手牌，拍位清空。
- 点击未锁定魔物拍后，该玩家拍位获得对撞目标，该魔物拍显示已锁定。
- 当一个魔物全部拍被锁定后，后续已填入玩家拍可以选择魔物本体。
- 未填入拍位的手牌不能直接选择魔物拍或魔物本体。
- 结束结算继续使用现有 `ResolveBeatRound`。
