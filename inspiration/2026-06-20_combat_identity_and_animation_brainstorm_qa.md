## 目标

开发者提出：当前《剑与黑塔》的战斗模式仍与《杀戮尖塔》非常类似，希望进行一场头脑风暴，把卡牌战斗变得更有自身特色，同时为玩家提供更强的战斗连贯性与动画演出爽感。

本轮讨论暂不直接进入实现，目标是先明确：

- 哪些现有设计导致战斗仍像传统 Roguelike 卡牌。
- 哪些《剑与黑塔》已有设定可以自然长出差异化。
- 是否需要新增轻量战斗层，让规则本身支持“动作连贯”和“漫画演出”，而不只是给现有规则套动画。
- 后续是否应把收束结果同步到正式 `design/` 文档和实现计划。

## 当前观察

### 仍然像《杀戮尖塔》的部分

当前新版基线虽然已经从旧“连锁层数 / 技能牌 / 卡牌包 / 3/5/8 阈值”迁移到“彩能 / 五色附魔 / 武器卡池 / 终结牌”，但基础战斗骨架仍然接近传统 Roguelike 卡牌：

- 每回合抽 5 张牌。
- 基础 3 行动点。
- 玩家回合逐张打出手牌。
- 魔物意图预告下一次行动。
- 玩家回合结束弃掉手牌，清空未用行动点和彩能。
- 通过奖励加牌、附魔和遗物扩展构筑。

这些规则稳定、可读，但如果只在其上增加数值和动画，玩家仍可能把它理解为“类似《杀戮尖塔》，只是多了颜色资源”。

### 已经具备差异化潜力的部分

文档中已有几条很强的原创线索：

- 左轮剑和机械臂不是装备数值，而是角色动作身份与卡池入口。
- 行动牌生成彩能，终结牌消耗彩能，是“前序动作铺垫 -> 爆发兑现”的结构。
- 五色不是职业标签，而是卡牌用途改写器。
- 视觉方向已经明确希望战斗像一页由玩家出牌驱动的黑白漫画，包含冲击格、速度线、局部特写、拟声字、跨格打击和命中停顿。
- 产品支柱强调“彩能 / 色彩 combo 爽感”和“画面演出与终结牌打击感”的优先级高于内容数量和平衡严谨度。

### 关键矛盾

当前设计目标想要“玩家自己驱动漫画战斗”，但规则层仍主要在处理：

- 能不能打出这张牌。
- 产生多少彩能。
- 消耗哪些彩能。
- 追加哪些颜色结算。

它还缺少一个中层机制来回答：

- 这些牌为什么连成一套动作。
- 左轮剑和机械臂如何在同一回合里形成可感知的衔接。
- 玩家为什么会觉得自己不是在逐张点卡，而是在编排一段战斗。
- 终结牌前的行动为什么会自然形成动画蓄势。

## 已确认的初步取向

### Q1：本次战斗差异化改动幅度应多大？

Codex 提出三个方向：

- A. 保守增强：保留现有 3 行动点 / 抽 5 / 回合制卡牌骨架，只把彩能槽、终结牌、漫画演出和连招反馈做得更爽。
- B. 中度重构：保留卡牌构筑与回合制，但新增一个轻量“动作连段 / 姿态 / 武器衔接”层，让出牌顺序直接生成连贯动作和演出收益。
- C. 激进重构：弱化传统回合制，改成更像“编排一套动作队列后集中演出 / 结算”的战斗模式。

开发者最初选择：B。随后修正为：C。

当前理解：

- 本轮可以弱化传统“玩家完整回合 -> 魔物完整回合”的结构。
- 战斗应更强调玩家与魔物之间的拼刀、格挡、反制和节奏交换。
- 新方向可能影响抽牌节奏、出牌时机、魔物意图、回合边界、卡牌目标选择和动画结算方式。
- 仍应尽量保留《剑与黑塔》已经确认的高价值资产：彩能、五色附魔、武器卡池、左轮剑 / 机械臂身份、终结牌爆发和黑白漫画演出。
- 需要警惕范围膨胀：如果采用 C，后续正式设计可能需要拆成“战斗节奏核心重构”和“卡牌 / 彩能 / UI 适配”两个阶段，而不是一次性全量实现。

### Q1 修正：为什么改为 C？

开发者补充：希望弱化传统回合制，让玩家与敌人之间更强调拼刀、格挡的节奏。

当前解读：

- 差异化不应只来自卡牌资源命名或终结牌动画，而应来自战斗进行方式。
- 玩家不只是轮到自己时连续打牌，还应经常根据魔物攻势做出挡、拼、闪、反或抢攻式选择。
- 魔物不只是回合末结算伤害的对象，而应成为推动节奏的对手。
- “拼刀 / 格挡”可以成为《剑与黑塔》区别于《杀戮尖塔》的核心战斗记忆点。

## 待继续讨论的问题

下一步需要继续确认：弱化传统回合制后，玩家与魔物的节奏关系应采用哪一种结构。

旧待选方向曾包括：

- 交替武器：左轮剑与机械臂来回衔接会变强，突出主副武器配合和动作连贯。
- 同武器专精：连续使用同一武器会叠出专属节奏，突出左轮剑连斩 / 弹幕或机械臂防反 / 控制。
- 动作类型顺序：按“启动 -> 控制 / 防御 -> 爆发 / 终结”的顺序出牌会变强，不强制武器交替，最容易教学和兼容现有卡牌。

在开发者修正为 C 后，上述问题需要后移。更优先的问题变为：

- 是采用“敌我交替行动”的短节拍结构？
- 是采用“敌人攻势倒计时，玩家在窗口内出牌应对”的半实时压力结构？
- 还是采用“玩家先编排 2-4 张牌，然后与魔物意图同时结算”的拼刀队列结构？

### Q2：拼刀、格挡节奏采用哪种敌我关系？

Codex 根据 Q1 修正后的 C 方向，提出三种结构：

- A. 敌我短节拍交替：玩家打一张牌，魔物推进 / 反应一步；战斗像你来我往的回合棋，最清楚、最可控。
- B. 攻势倒计时窗口：魔物攻击有倒计时或蓄力条，玩家在窗口内出牌、格挡、拼刀或打断；更有压迫感，但更接近半即时，需要 UI 和节奏设计更强。
- C. 同时编排后对撞：玩家先编排 2-4 张牌形成动作队列，然后与魔物意图同时结算，出现拼刀、破防、格挡、反击和终结演出；最有“分镜战斗”特色，也最适合做爽感演出。

开发者选择：C。

当前理解：

- 战斗高频决策从“本回合逐张即时结算”转向“先编排一段动作，再与魔物动作对撞”。
- 卡牌不只是单次效果按钮，而是动作队列中的一个镜头 / 招式。
- 魔物意图需要从单个回合行为扩展为可预读的动作序列或攻势片段。
- 拼刀、格挡、破防、反击和终结可以在队列结算阶段成为漫画分镜演出的自然节点。
- 这条路线最能支撑“玩家自己驱动漫画战斗”的目标，但也会显著影响规则层、UI 预览、动画事件和教学节奏。

### Q3：玩家应知道多少魔物动作？

Codex 提出三种预告粒度：

- A. 完整预告：魔物本轮 2-4 个动作全部公开，例如“斩击 -> 蓄力 -> 重击”。玩家像解一段战斗谜题，公平、策略强。
- B. 部分预告：只公开关键动作或类型，例如“第 2 拍有重击”，其余模糊。保留紧张感，但可读性更难。
- C. 意图轮廓：只显示整体倾向，例如“猛攻 / 防守 / 蓄力 / 控制”。玩家靠通用应对，不做精确对拍，节奏更快。

开发者选择：正常情况下完整预告；部分精英或 Boss 的技能、被动效果可以隐藏其中的动作。

当前理解：

- 普通战斗和基础教学应采用完整动作预告，保证玩家能公平读谱和编排。
- 隐藏动作不应成为默认规则，而应作为精英、Boss、特殊被动或特定技能带来的高级压力。
- 精英 / Boss 的隐藏动作需要有明确来源和可读提示，例如“黑雾遮蔽第 3 拍”“Boss 被动：末段动作隐藏”。
- 隐藏动作可以服务压迫感、记忆学习和 Boss 个性，但不能让失败变成不可复盘。
- UI 基准应先支持完整动作序列显示，再支持少量动作以问号、轮廓或特殊图标隐藏。

### Q4：玩家动作队列长度如何确定？

Codex 提出三种队列长度方向：

- A. 固定 3 拍：每轮最多编排 3 张牌，对应清楚的“三格漫画分镜”，适合 MVP 和教学。
- B. 2-4 拍动态：基础 3 拍，但可因武器、卡牌、黄色、遗物扩到 4，或因状态压缩到 2，更有成长空间。
- C. 按行动点自由填充：只要行动点够就能放更多牌，保留传统卡牌构筑爆发感，但容易回到《杀戮尖塔》式长回合。

开发者选择：不同武器有不同长度。初始武器左轮剑和机械臂均为 3，队列长度可以随着状态、遗物等变化。一拍固定出一张牌；但卡牌自身可以有多段数，也就是一拍可以产生多次攻击或格挡行为。魔物动作也遵循类似原则，敌人卡牌 / 动作自身也可以包含多段攻击或格挡行为。

当前理解：

- “拍”是编排层的最小队列格，一拍固定放置 1 张玩家卡牌或 1 个魔物动作。
- “段”是结算层的内部动作次数，一张牌可以在同一拍里产生多段攻击、多段格挡、多段射击或多次护盾判定。
- 队列长度成为武器身份的一部分：不同武器可以天然拥有不同节拍长度或改变节拍的方式。
- 初始 MVP 中左轮剑和机械臂都可以先使用 3 拍，降低教学复杂度。
- 后续状态、遗物、Boss 机制、武器升级或特殊卡牌可以扩展 / 压缩本轮节拍数。
- 多段牌能保留左轮剑弹幕、机械臂连续格挡等爽感，同时不会让编排层失控变成长回合。
- UI 需要区分“第几拍”和“本拍内几段”，否则玩家会混淆队列长度与多段结算。

### Q5：每一拍敌我对撞的核心判定关系是什么？

Codex 提出三种方向：

- A. 动作类型克制：攻击克蓄力，格挡克攻击，破防克格挡，闪避克重击等。直观、有拼刀感，但要维护一套类型表。
- B. 攻防数值对撞：双方这一拍各自产生攻击值 / 格挡值，比较后结算伤害、破防、反击。策略清楚，但可能变成算数题。
- C. 类型为主，数值为辅：先看攻击 / 格挡 / 破防 / 蓄力 / 终结等动作类型决定大结果，再用伤害、防御、多段数和彩能影响具体收益。

开发者选择：C。

补充设定：

- 攻击动作类型分为斩击、钝击和弹射。
- 左轮剑的一般攻击行动牌多为斩击。
- 左轮剑使用枪形态的终结牌偏弹射。
- 机械臂大部分攻击偏钝击。
- 不同魔物对不同攻击类型有三档抗性：暂称“抵抗 / 无抗性 / 弱点”，后续可换成更统一、更适合 UI 的命名。
- 非攻击动作类型分为格挡、闪避。
- 一张行动牌可以混合包含攻击动作与非攻击动作，例如先格挡再钝击，或闪避后斩击。

当前理解：

- 动作类型承担“拼刀 / 格挡 / 闪避 / 打中弱点”的战斗可读性。
- 数值仍然重要，但不再是第一层判定；它服务伤害、防御、破防程度、多段收益和彩能产出。
- 武器身份可以通过动作类型自然区分：左轮剑更偏斩击 / 弹射，机械臂更偏钝击 / 格挡。
- 魔物设计可以从传统意图扩展为“动作序列 + 攻击类型抗性 + 特殊隐藏动作”。
- 卡牌数据需要支持一张牌包含多个动作片段，每个片段有动作类型、数值、段数、目标和可能的彩能变化。
- 开发者后续澄清：“爆炸”是笔误，可以忽略。攻击类型暂时只保留斩击、钝击和弹射。

### Q6：格挡和闪避如何限制？

Codex 原本提出三种方向：

- A. 像普通牌一样占拍且有数值：格挡牌放进某一拍，对应该拍防住攻击；闪避牌放进某一拍，躲开特定攻击。
- B. 可覆盖相邻拍：格挡 / 闪避放在一拍，但能覆盖前后相邻的一小段窗口。
- C. 姿态型持续效果：打出格挡 / 闪避牌后，接下来数拍处于防御 / 闪避姿态，直到被打破或队列结束。

开发者选择：C，并澄清概念。

概念修正：

- 不使用“格挡牌”“闪避牌”这种说法。
- 攻击、格挡和闪避统称为“动作”。
- 一张行动牌可以混合包含攻击、闪避、格挡等多种动作。
- 某一拍的行动牌进行格挡或闪避动作时，会按照动作顺序抵挡或躲开对应魔物那一拍中的动作。
- 如果敌我双方在同一对撞位置都是抵挡或防御类动作，则这些动作直接互相消耗，不产生额外伤害。

当前理解：

- 卡牌是动作容器；动作才是对撞结算的最小规则单位。
- “一拍一张牌”不等于“一拍一个动作”。一张牌内部可能包含多个动作片段，按顺序参与本拍对撞。
- 格挡和闪避不应作为独立牌类，而应作为行动牌内部的动作片段或动作标签。
- UI 需要显示卡牌在该拍内的动作串，例如“格挡 -> 钝击”或“闪避 -> 斩击 x2”。
- 规则文本需要区分“行动牌”“动作”“攻击动作”“非攻击动作”“动作段数”，避免玩家混淆。

### Q7：一拍内双方动作数量不同，剩余动作如何处理？

Codex 提出三种方向：

- A. 剩余动作直接命中 / 生效：例如玩家“格挡 -> 斩击”，魔物只有“斩击”，格挡抵消后玩家斩击命中。
- B. 每拍只对撞一次，剩余段数折算加成：减少复杂动画，但会削弱多段动作的拼刀感。
- C. 剩余动作按目标防御 / 姿态继续结算：如果对方还有持续格挡 / 闪避姿态则继续消耗，否则命中。

开发者选择：A。

当前理解：

- 一拍内按双方动作顺序逐个对撞。
- 当一方动作串先耗尽，另一方剩余动作直接对目标生效。
- 这会强化“动作数量差”“动作顺序”和“多段牌”的意义。
- 例如“格挡 -> 斩击”对“斩击”时，格挡抵消魔物斩击，玩家剩余斩击命中魔物。
- 这能制造清楚的漫画演出：挡住、空拍、反击、追击。
- 风险是多段动作可能过强，需要用卡牌费用、彩能、动作威力、魔物多段防御或抗性控制。

### Q8：格挡和闪避分别如何应对攻击？

Codex 提出三种方向：

- A. 格挡挡多数攻击，闪避躲重击 / 弹射：格挡稳定，闪避偏针对，容易教学。
- B. 格挡克斩击 / 钝击，闪避克弹射 / 重击：更强调攻击类型，但可能让魔物设计更复杂。
- C. 格挡按数值挡，闪避按次数躲：格挡适合挡少数高伤动作，闪避适合躲多段低伤动作，和动作段数强绑定。

开发者提出正式方向：

- 格挡能够稳定降低受到的伤害。
- 格挡公式为：敌方攻击动作数值 - 己方格挡动作数值；反过来魔物格挡玩家攻击时同理。
- 闪避默认 50% 概率完全不受伤害。
- 闪避概率可以通过武器、遗物、卡牌效果等提升。
- 后续可以设计部分卡牌在闪避成功后获得额外动作收益。

当前理解：

- 格挡是稳定、可预测、低波动的防御动作。
- 闪避是高波动、高上限的防御动作，适合与概率提升、成功触发、反击动作联动。
- 格挡和闪避都不是牌类，而是行动牌或魔物动作内部的动作片段。
- 格挡动作需要有数值；闪避动作需要有概率，且可能有成功 / 失败分支。
- UI 需要在编排阶段显示格挡预计减伤值、闪避基础概率、构筑修正后的概率，以及闪避成功后可能触发的后续动作。
- 闪避随机性可能影响失败复盘，需要通过概率显示、可提升渠道和高风险高收益定位控制。

### Q9：新战斗结构中彩能从哪里来？终结牌如何释放？

Codex 提出三种方向：

- A. 卡牌打出时立刻生成：保持现有规则，行动牌放入队列就获得彩能。简单，但和“对撞结果”关系弱。
- B. 动作成功后生成：攻击命中、格挡有效、闪避成功后才生成彩能。最贴合拼刀手感，但可能让失败回合更雪崩。
- C. 放入队列给基础彩能，成功动作给额外彩能：保留稳定性，同时奖励打中、挡住、闪避成功。

开发者选择：B。

补充设定：

- 只有成功造成伤害、格挡有效或闪避成功时生成彩能。
- 终结牌不再和普通行动牌一样直接放入动作队列逐拍打出。
- 编排阶段可以放一张终结牌进入“终结槽”。
- 每当当前彩能满足终结牌释放条件时，玩家可以手动点击终结牌释放。
- 终结牌释放不受自方拍数限制。
- 终结牌释放时应有漫画分镜切入演出。

当前理解：

- 彩能变成“成功动作的战斗余势”，而不是“出牌即得的抽象资源”。
- 行动牌内部动作是否成功，会直接影响彩能生产和后续终结牌时机。
- 格挡和闪避不只是防御选择，也可以成为彩能生产方式。
- 终结牌从“队列中的一拍”变成“队列外的爆发切入”。
- 终结槽让玩家可以提前声明本轮想要的终结目标，并在彩能达标瞬间选择切入。
- 这会显著强化漫画演出：普通动作对撞积累彩能，终结牌打断常规节奏插入大分镜。
- 需要后续确认：终结牌切入会暂停当前拍、插在两拍之间、插在当前动作串结束后，还是可在任何成功动作后立即释放。
- 需要后续确认：终结槽中的终结牌是否占手牌、是否能替换、未释放时是否弃掉，以及敌人是否也能拥有类似终结 / 大招槽。

### Q10：终结牌满足彩能后，释放时机如何确定？

Codex 提出三种方向：

- A. 成功动作后可立即切入：每次玩家造成伤害、有效格挡或闪避成功后，如果彩能达标，就高亮终结槽，玩家可立刻释放。
- B. 每一拍结算后可切入：本拍双方动作都结算完，如果彩能达标，玩家可释放终结牌。
- C. 整轮队列结算后可释放：3 拍全部对撞完再释放。

开发者选择：A。

当前理解：

- 终结牌是战斗中达标即切入的手动大招。
- 每次玩家成功造成伤害、有效格挡或闪避成功后，如果彩能满足终结牌条件，终结槽应进入可释放状态。
- 玩家可以选择立即点击释放，也可以继续让动作队列推进，等待更高彩能或更合适的时机。
- 终结牌释放不占自方拍数，不是普通动作队列中的一张行动牌。
- 终结牌切入应使用明显漫画分镜演出，形成“拼刀中突然拔枪 / 爆发”的节奏。
- 需要避免频繁弹窗打断。更合适的是终结槽高亮、慢动作窗口、按钮脉冲或短暂停顿，让玩家可选择但不被强制中断。

### Q11：终结槽中的终结牌未释放时如何处理？

Codex 提出三种方向：

- A. 本轮有效，轮末弃掉：编排时放入终结槽，若本轮没释放，随手牌一起弃掉。
- B. 可跨轮保留：放入终结槽后一直保留，直到释放或替换。
- C. 消耗资源保留：默认轮末弃掉，但某些武器、遗物、卡牌效果可保留终结槽。

开发者选择：C。

补充设定：

- 终结牌在一轮内只要满足条件，可以多次释放。
- 终结牌释放后不会被消耗。
- 本轮结束后，终结槽中的终结牌默认弃掉。
- 部分武器、遗物、卡牌或其他效果可以让终结槽保留。

当前理解：

- 终结槽默认是本轮战术位，而不是永久装备槽。
- 同一张终结牌可以在本轮内反复作为彩能兑现入口。
- 彩能生产效率越高，终结牌多次释放的爆发越强。
- 保留终结槽属于高价值能力，应由武器、遗物、卡牌效果或特殊状态明确提供。
- 这会让“生成彩能 -> 立即终结 -> 继续动作 -> 再生成彩能 -> 再终结”的节奏成为核心爽感之一。
- 平衡风险：多次释放终结牌可能压倒行动牌，需要通过彩能条件、终结牌数值、每轮释放上限或特定终结牌限制控制。

### Q12：终结牌一轮内可多次释放时，是否需要默认次数限制？

Codex 提出三种方向：

- A. 不设默认次数限制：只要能持续产彩能就能反复放，最大化爽感，靠彩能生产和平衡控制。
- B. 每张终结牌每轮默认 1 次：最安全，但会削弱连续大招感。
- C. 按终结牌单独定义上限：默认数据字段支持“每轮最多 N 次”，多数 1-2 次，少数弹幕 / 特殊终结可更多。

开发者选择：A。

当前理解：

- 默认规则不限制同一张终结牌在一轮内的释放次数。
- 只要玩家能通过成功攻击、有效格挡或闪避成功持续生成足够彩能，就可以反复释放终结牌。
- 连续终结释放是本方向的重要爽感来源，不应先被次数上限削弱。
- 平衡重点转向彩能产出效率、终结牌彩能需求、动作对撞风险、敌人防守 / 闪避 / 抗性，以及终结牌本身数值。
- 特定终结牌若未来需要限制，可以作为单卡例外，而不是全局默认。

### Q13：新结构中是否保留行动点？

Codex 提出三种方向：

- A. 保留行动点作为编排成本：每张行动牌仍有行动点费用，限制放进队列的牌强度；传统构筑兼容最好。
- B. 取消行动点，队列拍数就是主要限制：每拍一张牌，强牌靠动作数值、彩能和副作用平衡；更不像《杀戮尖塔》。
- C. 改成武器负载 / 体势资源：不用“行动点”这个传统名词，改成更贴动作的资源，例如体势、负载、专注；本质仍限制强牌。

开发者选择：暂时先使用 B，简化系统。

当前理解：

- 当前脑暴方向暂时取消行动点。
- 每轮可编排拍数成为行动牌数量的主要限制。
- 卡牌强度主要通过动作数值、动作数量、动作类型、是否多段、是否产生彩能、终结牌需求、负面副作用和敌人对撞风险来平衡。
- 取消行动点有助于摆脱传统“3 费打牌回合”的手感。
- 需要后续确认旧数据中的费用字段如何迁移：删除、保留为兼容字段，还是转化为动作强度 / 稀有度 / 队列限制。

### Q14：每轮玩家手牌如何获得和保留？

Codex 提出三种方向：

- A. 仍然抽 5 张：保留传统抽牌循环，但从 5 张里选 3 张编排 + 可放 1 张终结槽。
- B. 抽等于队列拍数 + 终结候选：例如 3 拍抽 4 张左右，减少无用选择，更像动作编排。
- C. 手牌常驻 / 刷新池：不再每轮弃光，而是保留一组可用招式，每轮用掉的替换。

开发者提出正式方向：

- 默认战斗开始时抽 5 张牌。
- 手牌跨回合保留。
- 默认每回合开始时抽 2 张牌。

当前理解：

- 新结构中不再默认每回合弃掉全部手牌。
- 手牌成为持续战术资源，玩家可以攒动作牌或等待合适魔物动作序列。
- 每轮开始抽 2 张让手牌逐步流动，而不是每轮完全刷新。
- 战斗开始抽 5 张提供初始编排空间。
- 需要后续确认手牌上限、回合结束是否弃掉已编排但未使用的牌、用过的行动牌进入弃牌堆的时机，以及终结槽未释放时的弃牌规则。

### Q15：每轮编排后，用过的行动牌如何处理？

Codex 提出三种方向：

- A. 结算后进入弃牌堆：最接近卡牌游戏循环，清楚稳定。
- B. 只有动作成功才弃掉，失败回手：减少挫败，但可能被滥用试错。
- C. 按卡牌标记决定：普通行动牌结算后弃掉，某些“姿态 / 准备 / 武器技”可回手或保留。

开发者选择：A。

当前理解：

- 已编排并完成结算的行动牌进入弃牌堆。
- 手牌可以跨回合保留，但本轮用掉的行动牌会离开手牌。
- 这保留了牌库循环、弃牌堆和构筑厚度的意义。
- 行动牌不因动作失败而默认回手，避免试错滥用。
- 特殊回手或保留效果可以后续作为单卡、遗物或武器特性设计，但不作为默认规则。

### Q16：手牌不足，无法填满队列时怎么办？

Codex 提出三种方向：

- A. 允许空拍：没牌就空着，对撞时该拍不行动。
- B. 自动补基础动作：空拍自动变成弱格挡或待机，避免完全挨打。
- C. 必须至少填 1 拍即可开始：队列不必填满，玩家主动选择短队列，少行动但更快进入结算。

开发者选择：A。

当前理解：

- 玩家可以留下空拍。
- 空拍在对撞时不产生玩家动作。
- 如果对应魔物拍有攻击动作，玩家可能直接承受剩余动作。
- 手牌管理、抽牌能力和保留手牌的规划价值提高。
- 这会让“默认每轮抽 2 + 手牌跨回合保留”成为重要节奏，而不是每轮必然打满 3 拍。

### Q17：魔物是否也使用“牌 / 动作队列”结构？

Codex 提出三种方向：

- A. 魔物用固定动作序列数据：不是抽牌，只按 AI / 遭遇数据生成 3 拍动作。
- B. 魔物也有卡组 / 抽牌：魔物也像玩家一样从牌库抽动作，变化更大，但复杂很多。
- C. 普通魔物固定序列，Boss / 精英有动作牌池：普通好控，Boss 更有变化和记忆点。

开发者提出正式方向：

- 魔物不需要有卡组和抽牌。
- 魔物每一拍使用固定的“卡牌”来表现。
- 后续魔物卡牌也可以作为战利品。

当前理解：

- 魔物没有玩家式牌库、抽牌、手牌和弃牌堆。
- 魔物每一拍仍以卡牌化 UI 表示，方便玩家读取动作类型、数值、段数和隐藏状态。
- 魔物动作卡由遭遇、AI 或行为序列直接生成 / 指定。
- 魔物卡牌可以作为内容资产复用：既是敌人动作表现，也是未来奖励、战利品或特殊卡牌来源。
- 这能统一玩家卡牌与魔物动作的视觉语言，同时避免给魔物引入完整卡组系统。

### Q18：玩家获得魔物卡牌作为战利品后，它是什么性质？

Codex 提出三种方向：

- A. 临时战斗牌：只在当前战斗或下一场战斗可用，像偷学一招，范围可控。
- B. 可加入牌组的正式牌：击败魔物后可能获得其动作卡，长期构筑空间最大，但内容和平衡压力大。
- C. 作为改造素材：不直接加入牌组，而是给已有行动牌附加一个魔物动作、抗性或特效。

开发者选择：B。

当前理解：

- 魔物卡牌后续可以作为正式牌加入玩家牌组。
- 魔物动作卡既是敌人的攻击 / 防御表现，也可能成为玩家战利品。
- 这会增强“击败魔物 -> 学到或夺取其动作”的收集感和构筑记忆点。
- 平衡上需要区分武器卡、魔物卡、终结牌、色彩附魔和遗物的来源与稀有度。
- 需要后续确认魔物卡牌是否可被色彩碎片附魔、是否属于某个武器、是否占用奖励三选一，以及是否有特殊限制。

### Q19：魔物卡牌加入玩家牌组后，是否能被色彩碎片附魔并生成彩能？

Codex 提出三种方向：

- A. 可以，和行动牌一致：规则统一，构筑空间最大。
- B. 不可以，魔物卡只提供动作：保持武器行动牌和彩能系统的核心地位。
- C. 部分可以：只有被“驯化 / 解析 / 染色”的魔物卡可以附魔，作为稀有成长或事件奖励。

开发者选择：A。

当前理解：

- 魔物卡牌作为正式牌加入玩家牌组后，也可以被色彩碎片附魔。
- 魔物卡牌在玩家使用时可以像行动牌一样，通过成功造成伤害、有效格挡或闪避成功生成彩能。
- 色彩可以改写魔物卡牌的用途，使魔物动作融入玩家构筑。
- 这强化了“夺取魔物招式 / 黑塔异常能力，再用色彩重新改写”的主题。
- 需要后续确认魔物卡牌是否归类为行动牌的一种，还是新增来源标签“魔物牌”；玩家可见卡牌类型不宜过多。

### Q20：新结构下，原来的“行动牌 / 终结牌”两类是否仍保留为玩家可见类型？

Codex 提出三种方向：

- A. 保留两类：行动牌负责放入拍位，终结牌负责放入终结槽。清楚，兼容现有设计。
- B. 改成动作牌 / 终结牌：把“行动牌”改名为“动作牌”，更贴合新系统。
- C. 三来源但两类型：类型仍是行动牌 / 终结牌，但行动牌有来源标签：武器牌、魔物牌、特殊牌。

开发者选择：A。

当前理解：

- 玩家可见基础卡牌类型继续保留行动牌和终结牌。
- 行动牌放入队列拍位，提供一个或多个动作。
- 终结牌放入终结槽，满足彩能条件后可手动切入释放。
- 魔物卡牌如果进入玩家牌组，应归入行动牌，而不是新增第三种基础类型。
- 魔物来源、武器来源和特殊来源可以通过标签、卡框标记或奖励来源表达。

### Q21：彩能由成功动作生成后，附魔颜色作用在哪一层？

Codex 提出三种方向：

- A. 作用于整张行动牌：这张牌所有成功动作都生成同一种颜色彩能。
- B. 作用于具体动作：一张多动作牌可以只有某个动作带色。
- C. 作用于第一次成功动作：附魔牌本轮第一次成功动作生成带色彩能，后续动作生成无色。

开发者选择：A。

当前理解：

- 色彩碎片仍然附魔整张行动牌。
- 该行动牌内所有成功造成伤害、有效格挡或闪避成功的动作，生成的彩能都继承这张牌的附魔颜色。
- 未附魔行动牌的成功动作生成无色彩能。
- 这延续现有“给一张行动牌染色”的构筑直觉，避免把颜色细化到动作片段导致 UI 过载。
- 多动作牌被附魔后可能生成多点同色彩能，强度需要通过动作数量、成功条件、卡牌稀有度和魔物对撞风险控制。

### Q22：成功动作生成彩能的数量如何计算？

Codex 提出三种方向：

- A. 每个成功动作固定 1 点：命中一次、有效格挡一次、闪避成功一次，各得 1 彩能。
- B. 按动作段数 / 强度不同：强动作或多段动作生成更多彩能。
- C. 卡牌定义产能：每张行动牌写明每类成功动作生成多少彩能。

开发者选择：A。

当前理解：

- 每个成功动作固定生成 1 点彩能。
- 成功造成伤害、格挡有效或闪避成功都算成功动作。
- 彩能颜色来自该行动牌附魔；未附魔行动牌生成无色彩能。
- 一张多动作行动牌如果有多个动作成功，可以生成多点彩能。
- 该规则清楚、可预览，并能让玩家直接理解“动作串成功越多，终结越快”。

### Q23：彩能上限是否保留 6？

Codex 提出三种方向：

- A. 保留 6：延续现有设计，终结牌频率可控。
- B. 提高到 8 或 10：多动作成功后产能更高，需要更大槽位，爆发空间更大。
- C. 取消固定上限，轮内无限但轮末清空：最爽，但终结牌多次释放会非常难控。

开发者选择：A。

当前理解：

- 彩能基础上限继续保留 6。
- 多动作成功可以更快填满彩能槽，但不会无限膨胀。
- 满彩能后如果玩家不释放终结牌，后续成功动作可能浪费彩能。
- “成功动作产能 -> 满槽 -> 手动终结切入 -> 继续对撞产能”的节奏成为核心循环。
- 后续突破 6 点上限仍可作为遗物、状态、武器或特殊效果，但不进入默认规则。

### Q24：彩能在一轮结束后如何处理？

Codex 提出三种方向：

- A. 轮末清空：延续现有规则，强调本轮动作成功后即时终结。
- B. 跨轮保留：更容易放终结，但节奏压力小很多。
- C. 默认清空，但终结槽 / 遗物 / 状态可保留部分：和现有高价值保留规则一致。

开发者选择：C。

当前理解：

- 彩能默认在一轮结束后清空。
- 少数终结槽效果、遗物、状态、武器或特殊卡牌可以保留部分或全部彩能。
- 保留彩能仍属于高价值能力，需要清晰标注来源和持续时间。
- 默认节奏仍然是“本轮拼刀攒势，本轮找机会终结”。

### Q25：多魔物战斗中，玩家队列如何对撞多个魔物？

Codex 提出三种方向：

- A. 每拍选择一个主要目标：该拍行动牌主要和目标魔物的同拍动作对撞；其他魔物动作照常结算。
- B. 所有魔物共享一条合并攻势线：系统把多个魔物动作合并成 3 拍威胁，玩家对撞这条总攻势。
- C. 玩家每拍可分配目标，群体 / 多段动作可覆盖多个魔物：最完整，能保留多魔物策略，但 UI 和结算复杂度最高。

开发者选择：A，并做调整。

补充设定：

- 每一拍可选择一个魔物的任意一拍作为动作对撞目标。
- 玩家并不只能对撞“同序号”的魔物拍。
- 当一个魔物的所有拍数都已经被设为对撞目标，己方剩余拍数可以直接以该魔物本体为目标出牌。

当前理解：

- 玩家每个拍位需要同时选择一张行动牌和一个对撞目标。
- 对撞目标可以是“某个魔物的第 N 拍动作”，也可以在该魔物所有拍都已被接住后，改为“魔物本体”。
- 这让玩家可以主动拆解最危险的魔物动作，例如用第 1 拍格挡魔物第 3 拍重击。
- 对撞目标选择本身成为战斗策略：先拆招，再追击本体。
- 直接攻击本体的条件是该魔物本轮所有拍位都已被己方拍位指定为对撞目标。
- 多魔物战中，未被对撞的魔物动作仍需要结算，可能直接命中玩家或执行其效果。
- UI 需要清楚显示每个玩家拍位连到哪个魔物拍位，避免目标线混乱。

### Q26：多个玩家拍位是否可以锁定同一个魔物拍？

Codex 提出三种方向：

- A. 不允许重复锁定同一魔物拍：每个魔物拍最多被一个玩家拍接住。规则清楚，避免堆火力。
- B. 允许重复锁定并按顺序围攻：多个玩家拍可以连续压制同一个魔物动作。
- C. 默认不允许，特殊卡牌 / 遗物允许：基础规则清楚，同时给未来“夹击、连携、追击”留空间。

开发者选择：A。

当前理解：

- 基础规则下，每个魔物拍最多只能被一个玩家拍位指定为对撞目标。
- 玩家不能用多个拍位重复锁定同一个魔物动作。
- 这保证多魔物和多拍动作有真实分配压力。
- 如果一个魔物有 3 拍动作，玩家通常需要用 3 个己方拍位全部接住，才获得直接攻击该魔物本体的条件。
- 未来若需要“夹击 / 连携 / 追击”效果，可以作为特殊规则单独设计，但不进入默认规则。

### Q27：未被玩家对撞的魔物拍什么时候结算？

Codex 提出三种方向：

- A. 按魔物原本拍序在队列中结算：玩家接住的拍位发生对撞，没接住的魔物拍仍按它自己的顺序打玩家。
- B. 玩家队列结束后统一结算：未处理的魔物动作最后一起打过来。
- C. 每个玩家拍后结算一个未处理魔物拍：节奏更交替，但规则更难解释。

开发者选择：A。

当前理解：

- 魔物攻势是完整存在的动作序列。
- 玩家拍位只是选择截住 / 对撞其中若干魔物拍。
- 未被截住的魔物拍仍按魔物自身拍序正常结算。
- 这强化读谱和拆招：玩家需要决定哪些动作必须接，哪些可以承受、闪避或留给防御资源。
- UI 需要在编排时明确标出“已接住 / 未处理 / 可直接攻击本体”的魔物拍状态。

### Q28：玩家某拍直接以魔物本体为目标时，行动牌内部的格挡 / 闪避动作如何处理？

Codex 提出三种方向：

- A. 非攻击动作无效 / 跳过：直接打本体时只结算攻击动作。
- B. 非攻击动作转为准备收益：格挡给自身防御，闪避给本轮闪避率 / 下一动作加成，然后攻击命中本体。
- C. 不能用含格挡 / 闪避的混合牌直接打本体：避免复杂，但限制太硬。

开发者选择：A。

当前理解：

- 当玩家行动牌直接以魔物本体为目标时，只结算该牌内部的攻击动作。
- 该牌内部的格挡、闪避等非攻击动作跳过或不产生效果。
- 混合牌更适合用于拆招对撞。
- 纯攻击或高攻击段数牌更适合在敌人所有拍被接住后直接追击本体。
- UI 需要在直接瞄准本体时提示哪些动作会生效、哪些动作会被跳过。

### Q29：魔物对攻击类型的三档抗性如何命名？

Codex 提出三组玩家可见名称：

- A. 抵抗 / 标准 / 弱点：直白，最容易懂。
- B. 钝化 / 中性 / 破绽：更有动作味，但“钝化”可能难懂。
- C. 低效 / 正常 / 克制：偏规则表达，清楚但少一点世界观味。

开发者选择：A。

当前理解：

- 攻击类型抗性三档命名为：抵抗 / 标准 / 弱点。
- 三档分别作用于斩击、钝击、弹射三类攻击动作。
- 该命名直白，适合降低新战斗系统的信息负担。

### Q30：抗性三档如何影响伤害？

Codex 提出三种方向：

- A. 简单倍率：抵抗 50%，标准 100%，弱点 150%。清楚、好调参。
- B. 加减固定值：抵抗 -X，弱点 +X。低数值阶段好控，但后期不够自然。
- C. 影响彩能而不是伤害：抵抗少产彩能，弱点多产彩能，伤害保持卡牌数值。

开发者选择：A。

当前理解：

- 抗性三档先采用简单倍率。
- 初始建议为抵抗 50%，标准 100%，弱点 150%。
- 该倍率作用于斩击、钝击、弹射攻击动作造成的伤害。
- 彩能生成仍按“成功动作固定 1 点”处理，不因抗性档位变化，除非后续单卡或遗物写明。

### Q31：终结牌伤害是否受攻击类型抗性影响？

Codex 提出三种方向：

- A. 受影响：终结牌也标攻击类型，例如左轮剑枪形态终结是弹射，打弱点更爽。
- B. 不受影响：终结牌是彩能爆发，避免大招被抗性削弱。
- C. 部分受影响：终结牌有“武器伤害段”受抗性，“彩能追加段”不受抗性。

开发者选择：A。

当前理解：

- 终结牌也需要标记攻击类型。
- 终结牌造成的攻击伤害受目标对斩击、钝击、弹射的抗性影响。
- 左轮剑枪形态终结牌可偏弹射，机械臂终结牌可偏钝击，其他终结牌按动作语义定义。
- 终结牌仍然是彩能爆发，但不会脱离动作类型与魔物抗性的核心系统。
- 这会让终结牌释放目标选择更重要：对弱点释放收益高，对抵抗释放可能需要依赖色彩追加效果或改换目标。

### Q32：玩家什么时候看到魔物攻击类型抗性？

Codex 提出三种方向：

- A. 战斗中常驻公开：魔物血条旁直接显示斩击 / 钝击 / 弹射三档。
- B. 悬停魔物查看：默认界面更干净，悬停显示三档抗性。
- C. 首次命中后揭示：更有探索感，但可能让玩家觉得第一次选错不公平。

开发者选择：B。

当前理解：

- 魔物抗性默认通过悬停魔物查看。
- 主界面不常驻显示完整抗性表，以免压垮动作队列和魔物意图信息。
- 但在玩家拖拽 / 瞄准行动牌或终结牌时，可以对当前目标临时提示对应攻击类型是抵抗、标准还是弱点。
- 这样既保持界面干净，又减少重复悬停成本。

### Q33：当前阶段继续细化到哪一层？

Codex 提出三种方向：

- A. 继续细化规则：例如状态、Boss 隐藏动作、终结牌颜色结算、奖励、失败条件等。
- B. 先收束成 2-3 个整体方案：把目前内容整理成几种可执行战斗设计方案，比较取舍后选一个。
- C. 直接进入设计草案：基于目前已确认内容，先写一版完整战斗重构设计，之后逐节审。

开发者选择：B。

当前理解：

- 已确认规则足够形成整体方案比较。
- 下一步应先从多个可执行方向里选定主方案，而不是继续无边界细化。
- 方案比较应关注：与《杀戮尖塔》的差异化、实现范围、MVP 可验证性、动画演出潜力、规则可读性和平衡风险。

### Q34：三拍拆招制是否会过于接近《废墟图书馆》？

开发者确认采用“方案 1：三拍拆招制”作为主方向，但提出新的差异化风险：当前战斗会与《废墟图书馆》非常像，需要继续寻找差异化。

当前风险判断：

- “预告敌方动作 -> 我方选择卡牌 / 动作 -> 双方动作对撞”的结构容易让熟悉《废墟图书馆》的玩家产生联想。
- 若只停留在拼点、对撞和敌方动作预告，差异化不足。
- 《剑与黑塔》的差异化应优先从已有核心中生长：彩能、色彩附魔、终结槽手动切入、武器拍数、魔物动作卡战利品、黑白漫画分镜，以及“成功动作产能”的即时爆发节奏。

待收束方向：

- 是否将差异化重点放在“彩能与终结槽即时切入”。
- 是否将差异化重点放在“每拍自由拆敌方任意拍 + 打空门追击”。
- 是否将差异化重点放在“魔物动作卡可夺取并附魔”。
- 是否将差异化重点放在“黑白漫画分镜与手动大招插入”的表现结构。

阶段结论：

开发者确认采用 Codex 推荐的正式核心卖点：

> 三拍拆招 + 成功产彩 + 终结分镜切入

该卖点用于同时区分《杀戮尖塔》和《废墟图书馆》：

- 区别于《杀戮尖塔》：战斗不再是玩家完整回合中连续打牌、敌人回合统一行动，而是玩家编排拍位去拆解魔物动作序列。
- 区别于《废墟图书馆》：重点不只是动作对撞 / 拼点，而是通过成功攻击、有效格挡和闪避成功生成彩能，并在彩能达标时手动切入终结牌漫画分镜。
- 《剑与黑塔》的战斗记忆点应是“拆招制造成功动作，成功动作积累彩能，彩能驱动终结牌在战斗中插入爆发演出”。

四个支撑点：

1. 三拍拆招：玩家每轮按武器拍数编排行动牌，每拍选择魔物的任意未锁定拍作为对撞目标；拆完某个魔物全部拍后，剩余拍可直接攻击本体。
2. 成功产彩：只有成功造成伤害、格挡有效或闪避成功时生成彩能；每个成功动作固定 1 点，颜色来自行动牌附魔。
3. 终结分镜切入：终结牌放入终结槽，不占拍；满足彩能后可在成功动作后手动释放，一轮内默认不限次数，释放时插入漫画分镜演出。
4. 动作夺取构筑：魔物每拍用卡牌表现，后续魔物卡牌可作为正式战利品加入玩家牌组，并可被色彩碎片附魔。

## 当前非结论

以下内容尚未确认，不能视为正式规则：

- 是否新增“姿态”“连段”“武器衔接”“动作队列”等正式术语。
- 新机制是否影响数值收益、彩能生成、终结牌消耗，还是只影响演出和少量奖励。
- 新机制是否进入 MVP，或先作为后续垂直切片方向。
- 抽 5、3 行动点、回合结束弃牌和魔物意图等传统骨架会被保留、弱化还是替换。
