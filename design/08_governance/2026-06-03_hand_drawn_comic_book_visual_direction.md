# 决策：采用手绘漫画书风格作为主视觉方向

日期：2026-06-03  
状态：已接受  
影响范围：体验 / 视觉 / UI / 演出

## 背景

此前视觉方向的主次关系不够清晰，后续效果图和实现可能过度走向氛围装饰，削弱本项目最需要验证的连锁 combo 爽感、终结牌演出和 UI 可读性。

在本轮 [[inspiration/2026-06-03_03_experience_mvp_visual_qa|03 Experience MVP Visual Q&A]] 中，开发者确认希望画面风格改为手绘漫画风格：整体画面像漫画书一样，战斗演出使用漫画分镜，让玩家有看漫画书的感觉。

## 选项

- 方案 A：继续使用多风格并列的视觉基调。
- 方案 B：改为“手绘漫画书风格”为主视觉，并优先用关键元素母版统一后续原型。

## 决策

采用方案 B：第一版 MVP 和后续视觉探索以“手绘漫画书风格”为主视觉方向。

整体画面应像一本可交互漫画，使用手绘线稿、网点阴影、纸张纹理、粗细变化明显的墨线、漫画格和夸张动作姿态。战斗演出应通过漫画分镜、速度线、冲击格、局部特写、拟声字、跨格打击和命中帧停顿来放大连锁层数提升、高连锁阈值达成和终结牌释放。

后续视觉原型应先对齐关键元素母版：简洁漫画剑客主角、简洁漫画恶魔敌人、血红行动牌、冷青 / 幽蓝技能牌、紫黑终结牌、同色系卡牌包，以及简单偏 Q 版的漫画遗物。

## 理由

手绘漫画书风格更直接服务本项目的核心产品目标：让玩家通过牌序规划和构筑打出值得记住、值得截图和分享的 combo 高光。漫画分镜天然适合表达命中瞬间、爆发伤害、连锁升级和终结牌兑现，也能在独立游戏制作范围内以 2D UI、分层贴图、简单动画和事件驱动演出实现。

将风格收束到手绘漫画书，可以避免整体画面过暗、装饰过重或过于氛围化，影响 Roguelike 卡牌游戏最重要的信息读取。

## 后续行动

- 生成第一版 MVP 关键界面效果图，作为后续 Godot + C# 实现时的视觉上下文。
- 在 [[design/03_experience/01_visual_direction|视觉方向]] 中持续细化漫画书风格的色彩、线条、分镜和特效准则。
- 在 [[design/03_experience/00_ui_ux|界面与交互]] 中验证漫画分镜演出不会移动或遮挡核心战斗 UI。
- 后续若重新引入强题材装饰，需要再次记录决策并验证可读性。
