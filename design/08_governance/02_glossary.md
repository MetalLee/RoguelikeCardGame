## 目的

统一项目内的术语、关键词和机制命名，避免文档、UI 和数据表使用不同表达。

## 基础术语

| 术语   | 定义                            | 关联文档                                                   |          |
| ---- | ----------------------------- | ------------------------------------------------------ | -------- |
| Run  | 从开局到胜利或失败的一整局游戏流程。            | [[design/01_core_gameplay/01_run_structure]]           | 单局结构     |
| 构筑   | 玩家在一局中通过卡牌、遗物、资源和路线选择形成的策略组合。 | [[design/01_core_gameplay/03_card_system]]             | 卡牌系统     |
| 敌人意图 | 敌人在下一次行动中将执行的行为预告。            | [[design/01_core_gameplay/02_combat_system]]           | 战斗系统     |
| 遗物   | 通常跨战斗生效、改变规则或提供被动收益的奖励。       | [[design/02_content_systems/03_relics_items_rewards]]  | 遗物、道具与奖励 |
| 普通遗物 | 精英战斗在卡牌包之外额外给予的基础遗物类型，主要提供稳定构筑收益或规则变化；第一版 MVP 只做 1 个普通遗物，后续完整版本再扩展为随机池。 | [[design/02_content_systems/03_relics_items_rewards]] | 遗物、道具与奖励 |
| 特殊遗物 | Boss 战斗额外提供的高影响遗物类型；Boss 提供 3 个候选，玩家选择 1 个获得，应能影响构筑、改变玩法或显著改变后续决策。 | [[design/02_content_systems/03_relics_items_rewards]] | 遗物、道具与奖励 |
| 事件   | 非标准战斗节点中的选择型遭遇。               | [[design/02_content_systems/04_events_and_encounters]] | 事件与遭遇    |
| 连锁层数 | 玩家通过行动牌和少数特殊效果积累的回合内战斗节奏资源，默认在玩家回合结束或使用终结牌后归零；部分遗物或特殊终结牌可保留部分或全部层数。 | [[design/01_core_gameplay/02_combat_system]] | 战斗系统 |
| 行动点 | 默认由行动牌消耗的每回合战斗资源，用于限制攻击和叠连锁节奏；基础上限为 3，每回合恢复当前行动点上限 + 额外行动点数，未使用行动点在玩家回合结束时清空，可被回费、减费、临时行动点增加、行动点上限提升或下回合额外行动点影响。 | [[design/01_core_gameplay/04_resource_economy]] | 资源与经济 |
| 防御 | 用于抵挡敌人伤害的短期资源；玩家回合结束时不清空，敌人行动后若仍有剩余，则在下一回合开始时清空。 | [[design/01_core_gameplay/02_combat_system]] | 战斗系统 |
| 行动牌 | 主要负责攻击和叠连锁层数的卡牌类型；默认消耗行动点，使用后 +1 连锁层数。 | [[design/01_core_gameplay/03_card_system]] | 卡牌系统 |
| 技能牌 | 默认不产生连锁层数的卡牌类型，主要负责防御、抽牌、检索、减费、回费、状态、控制或调整牌序。 | [[design/01_core_gameplay/03_card_system]] | 卡牌系统 |
| 终结牌 | 默认消耗所有当前连锁层数并兑现高连锁收益的卡牌类型；默认不消耗行动点；此前文档中“终结卡”指同一概念。 | [[design/01_core_gameplay/03_card_system]] | 卡牌系统 |
| 卡牌包 | 普通战斗后的基础卡牌奖励形式；玩家从行动牌包、技能牌包、终结牌包中选择 1 包打开，选择前只知道类型、不知道具体卡牌，再从打开后的 3 张同类型候选卡牌中选择 0-3 张加入卡组；每包至少包含 1 张高于普通稀有度的卡牌。第一版 MVP 采用固定可重复池，每类卡牌包固定显示 3 张奖励牌，允许重复加入同名牌。 | [[design/01_core_gameplay/03_card_system]] | 卡牌系统 |
| 卡组厚度 | 当前卡组包含的卡牌数量及其对抽到关键牌稳定性的影响；过厚可能导致启动牌、延展牌或终结牌更难被及时抽到。 | [[design/01_core_gameplay/04_resource_economy]] | 资源与经济 |
| 抽牌堆 | 玩家当前可抽取卡牌的牌区；抽牌堆不足时，默认将弃牌堆洗回抽牌堆继续抽。 | [[design/01_core_gameplay/02_combat_system]] | 战斗系统 |
| 弃牌堆 | 已使用、被弃掉或回合结束弃置的普通卡牌所在牌区；可在抽牌堆不足或特定效果下被洗回抽牌堆。 | [[design/01_core_gameplay/02_combat_system]] | 战斗系统 |

## 关键词候选

| 关键词 | 候选定义 | 关联文档 | 备注 |
| --- | --- | --- | --- |
| 回费 | 在当前回合恢复或获得行动点的效果。 | [[design/01_core_gameplay/04_resource_economy]] | 默认可使当前行动点超过基础上限，具体数值边界和显示方式待定。 |
| 减费 | 降低行动牌行动点消耗的效果。 | [[design/01_core_gameplay/04_resource_economy]] | 默认影响行动牌；是否影响特殊牌型待定。 |
| 过牌 | 抽取、检索或以其他方式获得更多可用手牌的效果，用于提高卡组运转能力。 | [[design/01_core_gameplay/03_card_system]] | 与单纯抽牌、检索的文本边界待定。 |
| 重洗 | 将弃牌堆洗回抽牌堆的牌区操作；默认在抽牌堆不足时发生，部分卡牌可强制触发。 | [[design/01_core_gameplay/02_combat_system]] | 需要明确 UI、动画和是否随后抽牌 / 检索。 |
| 临时行动点增加 | 在当前回合临时增加可用行动点的效果。 | [[design/01_core_gameplay/04_resource_economy]] | 与回费的文本边界待定；默认玩家回合结束时未用部分清空。 |
| 行动点上限 | 决定每回合基础行动点恢复量的资源上限，基础值为 3，可被技能或遗物提升。 | [[design/01_core_gameplay/04_resource_economy]] | 不作为当前行动点不可突破的绝对硬上限。 |
| 额外行动点 | 预先增加指定后续回合可恢复行动点的资源修正；每回合恢复量为当前行动点上限 + 当回合额外行动点数。 | [[design/01_core_gameplay/04_resource_economy]] | 需要显示来源、数量和持续到哪个回合。 |
| 保留 | 使手牌、防御、行动点或连锁层数突破默认清空规则并延续到指定时点的效果。 | [[design/01_core_gameplay/02_combat_system]] | 需要明确保留对象、数量或比例、持续到何时。 |

所有正式关键词必须补充定义、显示文本、图标需求和结算时机。
