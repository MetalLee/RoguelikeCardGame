# 决策：通用色彩附魔与 MVP 构筑原型

日期：2026-06-14  
状态：已接受  
影响范围：玩法 / 卡牌 / 资源 / UI / 平衡 / 内容

> 2026-06-15 更新：MVP 起始流程已取消起始卡牌自由选择。当前规则为保留主副武器选择，每把武器固定 6 张初始牌，主手 6 张全带，副手只带 4 张行动牌。
>
> 2026-06-21 更新：本决策中的五色附魔与三条构筑原型仍作为后续方向保留，但不再属于当前第一版 MVP 切片。当前 `combat-rework-beat` 分支优先验证“三拍拆招 + 成功产无色彩能 + 终结槽手动释放 + 武器卡牌三选一”，色彩碎片、行动牌附魔和五色终结转译延后。
>
> 2026-06-21 范围更新：当前 MVP 起始卡池进一步收束为每把武器固定 4 张，主副武器合计 8 张；左轮剑与机械臂 active 卡池合计 6 张，3 场固定遭遇用于验证最小闭环。

## 背景

[[design/08_governance/2026-06-13_sword_black_tower_color_energy_core|2026-06-13 彩能与武器卡池决策]] 曾确认《剑与黑塔》采用五色彩能、色彩碎片、行动牌附魔和终结牌消耗彩能作为基线。2026-06-21 三拍重构后，当前第一版 MVP 暂时收敛为无色彩能切片；本文保留为后续五色阶段决策依据。

如果红、黄、蓝、绿、紫只绑定固定卡牌类型，例如红色只服务攻击、蓝色只服务防御、绿色只服务治疗，色彩系统会退化为传统职业标签或属性标签，无法支撑“用颜色改写卡牌用途”的构筑空间。

## 决策

所有色彩都可以增强所有卡牌。色彩不是职业标签，而是卡牌用途的改写器。

- 行动牌被色彩碎片附魔后，打出时生成对应颜色的彩能。
- 行动牌自身也可以被附魔颜色增强，不只是生成彩能。
- 终结牌消耗彩能时，根据被消耗彩能的颜色逐一触发颜色增强。
- 同一颜色可以叠加增强，但五色阶段需要写明数值边界或硬上限。
- 红色、黄色、蓝色、绿色、紫色都能作用于攻击、防御、控制和终结牌。
- MVP 卡池以左轮剑和机械臂合计 20 个独立卡牌设计作为当前内容基线。
- MVP 需要实现主副武器选择：玩家可自由决定左轮剑 / 机械臂谁是主手；系统自动将主手固定初始池 6 张全部加入，并将副手固定初始池中的 4 张行动牌加入。
- 后续五色构筑验证目标从“至少 2 种路线”扩展为 3 种早期构筑：蓝绿重炮流、红色机械防反流、黄紫弹幕调色流。

## 结果

- 左轮剑可以走蓝绿生存爆发：重炮不只是伤害终结牌，也可以通过蓝色获得护甲 / 防御，通过绿色回复生命。
- 机械臂可以走红色输出防反：格挡、能量护幕等防御牌可以通过红色把本次获得防御的一部分转化为伤害。
- 黄紫可以支撑弹幕 / 多段 combo：子弹风暴、二连斩、回旋剑舞等多段牌可以通过黄色增加卡牌释放次数，通过紫色放大异常或最终效果。
- 后续五色阶段的卡牌池、附魔、奖励和 UI 预览都需要围绕“颜色改写用途”组织，而不是围绕“颜色对应牌型”组织。

## 风险

- 黄色额外释放次数可能形成循环触发或过长动画；后续黄色不提供抽牌、返还行动点或回能量。
- 紫色复制、翻倍和异常延长可能造成数值膨胀或循环触发。
- 绿色治疗和蓝色护甲 / 防御可能让玩家过度安全，削弱魔物压力。
- 颜色预览、终结牌结算拆解和战斗日志复杂度上升。
- 所有颜色都能作用所有牌型后，单卡说明和数据字段需要更清楚，否则玩家难以判断奖励价值。

## 后续行动

- 先以文档和数据结构支持通用色彩增强，具体数值进入试玩调优。
- 为黄色和紫色在数据层预留单轮触发上限、额外释放次数上限和复制链上限。
- 为蓝色和绿色的最终伤害 / 最终效果值转译比例建立可调参数。
- UI 需要提供行动牌附魔预览、终结牌颜色消耗预览、最终预估收益和结算日志拆解。
- 后续重构 `game/` 数据和规则层时，应把旧“技能牌 / 连锁阈值 / 卡牌包”字段迁移到“行动牌 / 终结牌 / 彩能 / 色彩结算”表达。
