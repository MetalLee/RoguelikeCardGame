# 决策：以连锁层数作为 combo 爽感兑现机制

日期：2026-06-02  
状态：已接受  
影响范围：战斗 / 卡牌 / 体验 / 平衡

## 背景

在 [[inspiration/2026-06-02_00_product_brainstorm_qa|00 Product 头脑风暴 Q&A]] 中，开发者进一步确认 combo 爽感的具体来源：玩家通过打出卡牌增加连锁层数，通过牌序规划叠高层数，并在高连锁下使用具备额外效果的卡牌造成巨额伤害和夸张动画反馈。

## 选项

- 方案 A：以自动连锁触发作为 combo 爽感来源，玩家主要负责组牌。
- 方案 B：以连锁层数作为战斗节奏资源，玩家通过牌序主动积累并用终结卡兑现。
- 方案 C：以长期状态叠加作为主要爆发来源，连锁仅作为辅助表现。

## 决策

采用方案 B：以连锁层数作为 combo 爽感兑现机制。

## 理由

该方案能把构筑和战斗操作连接起来。玩家不只是组出强卡组，还需要在战斗中判断牌序、资源、防御压力和爆发窗口，从而获得“我把这套 combo 打出来了”的参与感。

连锁层数也能形成清晰的内容生产框架：启动牌负责起步，延展牌负责维持连锁，终结卡负责高连锁爆发，稳定牌负责在未成型或高压回合维持可玩性。

连锁层数的生命周期采用“回合内有条件保留”规则：默认在回合结束或使用终结卡后归零；部分遗物或特殊终结卡可以保留部分或全部连锁层数。

终结卡数量不设置硬性限制。默认卡组和前期构筑推荐 1-2 张终结卡；后期当过牌、减费、回费、复制或连锁保留能力提升后，玩家可以加入更多终结卡。多终结卡构筑可以形成流派，但应承担更高构筑成本和卡手风险。

终结卡的高连锁额外效果可以包括伤害增加、暴击率增加、群体伤害、保留连锁、抽牌、赋予敌人异常状态、赋予自身增益，以及秒杀非 Boss 敌人等方向。效果数值可以是固定值，也可以根据连锁层数加算或乘算。

基础终结卡效果候选包括单体高伤害、群体伤害、暴击率提升、抽牌 / 回费；具体首版内容待其他设计维度确认后再定。

连锁阈值基准采用 3 / 5 / 8 三档。3 层用于让玩家理解连锁机制；5 层用于触发抽牌、回费等续航连锁效果；8 层用于卡组成型后的高光爆发，例如巨额群体伤害，或后续扩展中的秒杀非 Boss 敌人。连锁层数不设置硬上限，超过 8 层后仍可继续计数，具体收益曲线和 UI 表达后续定义。

## 后续行动

- 定义超过 8 层后的收益曲线、清空 / 保留规则和 UI 表达。
- 在卡牌系统中制定连锁相关卡牌模板。
- 验证不同阶段的合理终结卡密度，以及过牌能力对多终结卡流派的支撑程度。
- 验证单体高伤害、群体伤害、暴击率提升、抽牌 / 回费等候选终结卡的可读性、爽感和数值边界。
- 验证 3 / 5 / 8 三档阈值的触发频率、收益差距和玩家理解成本。
- 在视觉方向中验证高连锁爆发特效的可读性和制作成本。
