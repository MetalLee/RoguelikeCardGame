# 决策：以连锁层数作为 combo 爽感兑现机制

日期：2026-06-02  
状态：已替代  
影响范围：战斗 / 卡牌 / 体验 / 平衡

> 2026-06-13 更新：本决策中“连锁层数作为抽象机制”的部分已被 [[design/08_governance/2026-06-13_sword_black_tower_color_energy_core|确立《剑与黑塔》的彩能、色彩与武器卡池核心]] 替代。新版中使用「彩能」作为世界观命名和规则表达，不再保留 3 / 5 / 8 固定阈值、技能牌类型或卡牌包奖励结构。本文保留为历史决策记录。

## 背景

在 [[inspiration/2026-06-02_00_product_brainstorm_qa|00 Product 头脑风暴 Q&A]] 中，开发者进一步确认 combo 爽感的具体来源：玩家通过打出卡牌增加连锁层数，通过牌序规划叠高层数，并在高连锁下使用具备额外效果的卡牌造成巨额伤害和夸张动画反馈。

在 [[inspiration/2026-06-02_01_core_gameplay_brainstorm_qa|01 Core Gameplay 头脑风暴 Q&A]] 中，开发者进一步确认卡牌类型分为行动牌、技能牌和终结牌，并以卡牌类型决定默认连锁行为。

## 选项

- 方案 A：以自动连锁触发作为 combo 爽感来源，玩家主要负责组牌。
- 方案 B：以连锁层数作为战斗节奏资源，玩家通过牌序主动积累并用终结牌兑现。
- 方案 C：以长期状态叠加作为主要爆发来源，连锁仅作为辅助表现。

## 决策

采用方案 B：以连锁层数作为 combo 爽感兑现机制。

## 理由

该方案能把构筑和战斗操作连接起来。玩家不只是组出强卡组，还需要在战斗中判断牌序、资源、防御压力和爆发窗口，从而获得“我把这套 combo 打出来了”的参与感。

连锁层数也能形成清晰的内容生产框架：启动牌负责起步，延展牌负责维持连锁，终结牌负责高连锁爆发，稳定牌负责在未成型或高压回合维持可玩性。

连锁层数的生命周期采用“回合内有条件保留”规则：默认在玩家回合结束或使用终结牌后归零；部分遗物或特殊终结牌可以保留部分或全部连锁层数。

卡牌类型分为行动牌、技能牌和终结牌。默认规则为：行动牌主要负责攻击和叠连锁层数，消耗行动点，使用后 +1 连锁层数；技能牌主要负责防御、抽牌、检索、减费、回费、状态、控制或调整牌序，默认不消耗行动点且不产生连锁层数；终结牌需要满足自身连锁层数使用条件后才能打出，默认不消耗行动点，使用后消耗所有当前连锁层数，并产生强大爆发效果。少数卡牌和遗物可以通过明确效果突破默认规则，例如额外增加层数、消耗层数或保留层数。

基础行动点上限为 3。遗物或技能可以提高行动点上限；行动点上限决定每回合基础恢复量，而不是当前行动点不可超过的绝对硬上限。行动点可以被卡牌、遗物或其他效果继续增加，用于支持更长行动牌链条和更高连锁层数。某些卡牌可以预先增加下回合的额外行动点数；每回合恢复的行动点数 = 当前行动点上限 + 当前回合应获得的额外行动点数。

默认回合结算规则为：玩家回合结束时，未使用行动点清空，连锁层数清空，未使用手牌全部弃掉；防御不会在玩家回合结束时清空，而是在敌人行动后、下一回合开始时清空剩余防御。下一回合开始时抽 5 张牌，并按当前行动点上限 + 额外行动点数恢复行动点；无修正时恢复 3 点。少数技能牌、终结牌或遗物可以保留手牌、防御、行动点或连锁层数。

默认牌库循环规则为：抽牌堆不足时，将弃牌堆洗回抽牌堆继续抽。普通卡牌在同一场战斗中可以多轮循环使用；能否再次打出仍受行动点、手牌、连锁层数、终结牌使用条件和其他卡牌条件限制。部分卡牌效果可以强制将弃牌堆整体洗回抽牌堆，用于制造检索、循环或爆发回合。消耗牌、临时牌和特殊牌区留到后续卡牌模板阶段定义。

牌组构筑完善后，技能牌和终结牌可以通过回费和过牌提高卡组运转能力，支撑更长行动牌链条、更高连锁层数和更多终结牌兑现窗口。

默认初始牌组为 10 张牌：4 张 1 费普通攻击、1 张 0 费弱攻击、1 张 2 费强攻击、2 张基础防御技能牌、1 张过牌技能牌、1 张终结牌。2 张基础防御技能牌提供防御值并 +1 行动点，1 张过牌技能牌抽 2 张牌。该结构用于让玩家尽快理解行动牌叠层、技能牌防御 / 过牌、终结牌兑现的基础链路。

初始牌组中的终结牌至少需要 3 层连锁才能打出，定位为单体爆发型终结牌：3 层造成高额单体伤害，5 层连锁及以上造成翻倍伤害。该模板用于让玩家自然理解终结牌的使用条件，以及连锁层数对终结牌效果强度的影响。其他特殊效果类型的终结牌应作为闯关奖励、构筑转向或流派扩展内容出现。

行动牌费用与连锁增长分离。行动牌费用可高可低，但默认每打出 1 张行动牌只增加 1 层连锁。初始牌组中的行动牌采用强弱对比结构：4 张 1 费普通攻击、1 张 0 费弱攻击、1 张 2 费强攻击。0 费弱攻击帮助玩家更早触达 5 层连锁，但需要控制基础强度；2 费强攻击用于体现费用影响伤害和回合节奏，但默认仍只 +1 连锁。

初始卡组使用保守数值，方便后续通过奖励感到成长明显。具体伤害值、防御值和终结牌基础伤害不在当前头脑风暴阶段确定，留到最小原型阶段统一设定和测试。

终结牌数量不设置硬性限制。默认卡组和前期构筑推荐 1-2 张终结牌；后期当过牌、减费、回费、复制或连锁保留能力提升后，玩家可以加入更多终结牌。多终结牌构筑可以形成流派，但应承担更高构筑成本和卡手风险。

终结牌的高连锁额外效果可以包括爆发伤害、高额防御、伤害增加、暴击率增加、群体伤害、保留连锁、抽牌、回费、赋予敌人异常状态、赋予自身增益，以及秒杀非 Boss 敌人等方向。效果数值可以是固定值，也可以根据连锁层数加算或乘算。

基础终结牌效果候选包括单体高伤害、群体伤害、暴击率提升、抽牌 / 回费；具体首版内容待其他设计维度确认后再定。

连锁阈值基准采用 3 / 5 / 8 三档。3 层用于让玩家理解连锁机制；5 层用于触发抽牌、回费等续航连锁效果；8 层用于卡组成型后的高光爆发，例如巨额群体伤害，或后续扩展中的秒杀非 Boss 敌人。连锁层数不设置硬上限，超过 8 层后仍可继续计数，具体收益曲线和 UI 表达后续定义。

## 后续行动

- 定义超过 8 层后的收益曲线和 UI 表达。
- 定义行动点上限提升、当前回合行动点增加和下回合额外行动点的数值边界与 UI 表达。
- 定义普通重洗、强制重洗、消耗牌、临时牌和特殊牌区的文本模板与 UI 表达。
- 在最小原型阶段统一设定初始攻击、防御和终结牌伤害数值。
- 定义保留手牌、防御、行动点或连锁层数的例外效果定价。
- 在卡牌系统中制定连锁相关卡牌模板。
- 验证不同阶段的合理终结牌密度，以及过牌能力对多终结牌流派的支撑程度。
- 验证单体高伤害、群体伤害、暴击率提升、抽牌 / 回费等候选终结牌的可读性、爽感和数值边界。
- 验证 3 / 5 / 8 三档阈值的触发频率、收益差距和玩家理解成本。
- 在视觉方向中验证高连锁爆发特效的可读性和制作成本。
