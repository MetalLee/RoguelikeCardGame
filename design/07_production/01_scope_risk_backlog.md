# 范围、风险与待办池

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

记录项目范围边界、主要风险和待办池，避免想法无限膨胀。

## 范围分级

- 必须有：没有它就不能验证核心循环。
- 应该有：显著改善首版体验，但可用简化方案替代。
- 可以有：有价值，但不影响核心验证。
- 暂不做：当前阶段会分散资源或风险过高。

## 当前必须有

- 左轮剑 + 机械臂起始武器。
- 主副武器选择与自动起始牌组。
- 行动牌 / 终结牌两类卡牌。
- 三拍拆招战斗：玩家拍位、魔物拍位、动作对撞、空门直击。
- 行动牌放入空拍后立即进入指向目标；取消目标会退回手牌。
- 无色彩能上限 6 和每轮清空。
- 终结牌进入终结槽，满足无色彩能条件后可手动切入释放。
- 普通战斗后武器卡牌三选一。
- 精英普通遗物。
- Boss 简化奖励。
- 黑塔世界观化的 6 场固定遭遇。

## 当前暂不做

- 完整事件系统。
- 路线图。
- 商店。
- 休息节点。
- 完整货币经济。
- 五色色彩碎片附魔行动牌。
- 终结牌按消耗颜色结算追加效果。
- 完整武器解锁池。
- 传统装备掉落系统。
- 长期 Meta 成长。

事件系统是 MVP 战斗打磨后的下一阶段优先项，高于商店和路线图。

## 主要风险

- 拍位、对撞目标、空门目标和终结槽叠加后 UI 复杂度过高。
- 三拍编排与动作结算如果反馈不足，玩家难以理解伤害来源。
- 后续五色、附魔和武器卡池叠加后 UI 复杂度可能再次升高。
- 主副武器选择和自动起始牌组说明不清，导致玩家不理解初始卡组来源。
- 内容生产量超过独立团队能力。
- 碎片化叙事太隐晦，玩家无法形成情感连接。
- 旧行动点 / 技能牌 / 连锁阈值文档或 UI 残留会误导实现和试玩。
