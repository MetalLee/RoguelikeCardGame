# 试玩测试计划

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义第一版 MVP 和后续版本的试玩问题、观察指标和记录方式。

## 第一版 MVP 测试目标

用 10-15 分钟线性章节验证：

- 主副武器选择和自动起始牌组规则是否可理解。
- 三拍拆招、行动牌放入拍位和指向魔物拍位 / 魔物本体是否可理解。
- 动作对撞、格挡、闪避、空门直击的结算是否清楚。
- 无色彩能生成、上限 6 和每轮清空是否清楚。
- 终结槽、手动终结牌释放和漫画分镜切入是否爽快、可读。
- 普通战斗后“武器卡牌三选一”是否形成有效成长。
- 精英普通遗物是否可感知。
- Boss 通关结算是否让玩家期待后续阶段。
- 3 场固定遭遇是否完成教学递进。

## 关键观察问题

- 玩家是否能在第一场战斗理解如何把行动牌放入空拍并选择对撞目标。
- 玩家是否能理解成功造成伤害、有效格挡或成功闪避才生成彩能。
- 玩家是否知道轮末未使用彩能会清空。
- 玩家是否能解释某张终结牌为什么能打出或不能打出。
- 玩家是否能看懂终结牌消耗了多少无色彩能。
- 玩家是否能理解放入拍位后必须完成指向，否则行动牌退回手牌。
- 玩家是否理解左轮剑和机械臂的职责差异。
- 玩家是否在 Boss 前形成至少一个明确构筑方向。
- 玩家是否通过卡牌、遗物或魔物文本感知到黑塔世界观。

## 测试记录

- Run 种子。
- 主副武器选择和自动起始牌组结果。
- 每战胜负、轮数、剩余生命。
- 每战最高彩能数。
- 每战终结牌使用次数和平均消耗彩能数。
- 行动牌放入拍位后的取消次数、无效目标次数和空门目标次数。
- 战后卡牌候选和选择。
- 遗物触发次数。
- Boss 奖励反馈。
- 玩家口述困惑点。

## 通过标准

- 玩家无需开发者讲解即可完成第一场战斗。
- 玩家能在 3 局内理解拍位、对撞目标、无色彩能和终结槽关系。
- 玩家至少能说出一个自己主动形成的构筑方向。
- 玩家能解释一次失败原因。
- 玩家愿意重开尝试不同主副武器顺序、拍位编排或奖励选择。
