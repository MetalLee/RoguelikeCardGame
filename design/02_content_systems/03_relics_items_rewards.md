# 遗物、道具与奖励

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义长期奖励如何改变规则、强化构筑或制造路线选择。

## 奖励类型

- 卡牌奖励。
- 遗物或被动。
- 货币。
- 治疗。
- 最大生命变化。
- 卡牌升级、删除、复制、转化。
- 特殊地图或事件入口。

## 装备系统边界

当前没有添加装备系统的计划。本项目的长期奖励应优先通过卡牌、遗物、被动规则变化、路线资源和卡组编辑来服务构筑，而不是通过装备掉落和数值刷装来驱动留存。

## 遗物设计准则

- 好遗物应改变决策方式，而不只是无条件加数值。
- 触发频率越高，单次收益越应克制。
- 负面代价必须可被玩家理解和规划。
- 与卡牌原型绑定过深的遗物需要保证出现时机和奖励池合理。
- 精英战斗在卡牌包之外额外随机给 1 个普通遗物，作为高风险节点的稳定额外收益。
- 第一版 MVP 的遗物池采用极简小型教学池，只做 1 个普通遗物，优先服务精英战斗后的遗物奖励，让玩家能在普通战斗 4 和 Boss 战中感受到规则收益；该版本不需要完整普通遗物池或特殊遗物池。
- 当前阶段不继续确认这个普通遗物的具体效果；具体触发条件、数值和文本进入 MVP 开发过程中按“让玩家在普通战斗 4 和 Boss 战中感受到规则收益”的目标落地。
- 后续完整 Run 的阶段性 Boss 战斗额外提供 3 个特殊遗物候选，玩家选择 1 个获得；特殊遗物应能影响构筑、改变玩法或显著改变后续决策，而不是单纯数值增强。
- 第一版 MVP 的终点 Boss 击败后直接胜利，不发放 Boss 战后特殊遗物或卡牌包奖励。

## 奖励选择准则

- 奖励应提供构筑强化、补短板和转向三类可能。
- 奖励界面应清楚说明即时收益与长期影响。
- 奖励不应退化为纯数值装备替换；即使提供数值强化，也应服务卡组联动、连锁层数或终结卡兑现。
- 后续完整 Run 的 Boss 奖励还应包含稀有度更高的卡牌包，并和特殊遗物三选一共同形成阶段性构筑转折点。
