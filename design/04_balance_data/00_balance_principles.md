# 平衡原则

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义数值和平衡工作的判断标准。平衡不是让所有选择平均，而是让不同选择在不同条件下有合理舞台。

## 核心原则

- 选择应有场景差异，而不是永久最优。
- 高收益需要成本、条件、时机或风险。
- 平衡严谨度服务核心爽感，但低于彩能 / 色彩 combo 爽感、画面演出、内容数量和构筑深度。
- 新玩家可通关路径和高手优化路径可以不同。
- 强组合可以存在，但应需要构筑投入或承担短板。
- 高彩能终结牌可以造成夸张收益，但必须要求牌序规划、手牌条件、彩能投入、色彩配置或防御取舍。
- 默认规则下，回合结束会清空未使用彩能；任何保留部分或全部彩能的效果都属于高价值能力，需要被重点平衡。
- 终结牌密度应形成自然取舍：前期 1-2 张更稳定，后期只有在过牌、减费、回费、彩能生成或彩能保留能力足够时才适合加入更多终结牌。
- 多终结牌流派可以存在，且终结牌之间可以联动，但需要承担更高构筑成本和卡手风险。
- 终结牌数值可以采用固定值、随消耗彩能数量加算、随特定色彩数量加算或乘算；乘算收益必须有更高消耗、资源成本或使用限制。
- 秒杀非 Boss 敌人的终结牌可以存在，但必须排除 Boss，并通过彩能消耗、目标范围、稀有度或触发次数限制控制强度。
- 不再使用 3 / 5 / 8 固定阈值；终结牌条件和收益曲线由单卡定义。
- 彩能基础上限为 6；超过 6 的突破效果应重点控制，避免无限堆叠压倒其他决策。
- 彩能机制应奖励高手规划，但不能让未成型卡组完全失去基础战斗能力。
- 红、黄、蓝、绿、紫 5 种色彩都需要可用舞台，不能让某一颜色成为无条件最优。
- 黄色额外释放次数和紫色翻倍是高风险设计，需要优先监控失控组合。
- 色彩增强必须优先保证可读性，再追求复杂联动。玩家应能从 hover 预览、彩能槽颜色和结算日志理解收益来源。
- 随机性不应长期掩盖玩家决策影响。

## 色彩平衡边界

MVP 不追求完全平衡，但必须避免无限循环、不可读爆发和无条件最优色彩。所有色彩都可以增强所有卡牌，平衡重点应从“颜色是否对应牌型”转为“颜色改写用途后的收益是否可理解、可控、有代价”。

- 黄色额外释放次数需要硬上限。MVP 黄色附魔效果只增加卡牌释放次数，不提供抽牌、返还行动点或回能量；建议每张牌最多因黄色增加 1 次额外整牌释放，具体数值待调参。
- 紫色复制 / 翻倍 / 异常延长需要硬上限，且不能复制会再次触发紫色复制的完整结算链，避免无限循环。
- 绿色回复不能超过最大生命。群攻牌或多段牌吃绿色时，应以最终总伤害或最终效果值按比例转化，而不是每个小段独立满额回复。
- 蓝色护甲 / 防御遵守默认防御清空规则；若某张牌、遗物或色彩核心允许跨回合保留，必须作为稀有规则单独标注。
- 终结牌同时造成伤害、护甲 / 防御、治疗时，应使用“最终效果值”作为统一结算基准，但比例可以调低。
- 防御牌吃红色转伤时，应先以“本次获得防御的一定比例”计算，不直接用当前总防御，避免滚雪球过快。
- 多段牌与红 / 黄 / 紫的联动是高风险组合，需要重点试玩，包括二连斩、回旋剑舞、子弹风暴和任何复制 / 追加释放效果。
- 蓝绿重炮、红色机械防反、黄紫弹幕调色三条 MVP 构筑都应能通关固定线性 Run，但不要求三者在所有遭遇中效率完全相同。

## 平衡取舍边界

早期平衡目标是避免无解、误导和单一最优，而不是过早追求所有流派完全均等。若某个强 combo 能显著提升爽感和分享欲，但仍有构筑成本、出牌条件或风险，可以优先保留并观察。

## 平衡对象

- 卡牌费用、彩能生成、彩能消耗、效果和升级。
- 武器初始卡池和奖励卡池。
- 终结牌密度、抽到终结牌的稳定性和卡手概率。
- 彩能上限、清空规则、保留规则、固定 / 加算 / 乘算收益和终结牌效果类型。
- 色彩碎片出现顺序、附魔对象限制和色彩核心强度。
- 红 / 黄 / 蓝 / 绿 / 紫效果边界。
- 敌人生命、伤害和行为。
- 遗物触发频率和收益。
- 卡牌奖励池出现概率。
- 跳过 / 删除 / 转化卡牌是否会破坏卡组厚度取舍。
- 路线节点密度。
- 商店价格。
- 治疗和资源恢复。

## 调整优先级

1. 修复无解、误导或不可读问题。
2. 削弱无条件最优策略。
3. 增强有趣但缺少支撑的构筑。
4. 微调数值效率。
