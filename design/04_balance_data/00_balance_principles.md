# 平衡原则

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义数值和平衡工作的判断标准。平衡不是让所有选择平均，而是让不同选择在不同条件下有合理舞台。

## 核心原则

- 选择应有场景差异，而不是永久最优。
- 高收益需要成本、条件、时机或风险。
- 平衡严谨度服务核心爽感，但在产品取舍中低于连锁 combo 爽感、画面演出、内容数量和构筑深度。
- 新玩家可通关路径和高手优化路径可以不同。
- 强组合可以存在，但应需要构筑投入或承担短板。
- 高连锁终结卡可以造成夸张收益，但必须要求牌序规划、手牌条件、资源投入或防御取舍。
- 默认规则下，回合结束或使用终结卡会清空连锁层数；任何保留部分或全部连锁层数的效果都属于高价值能力，需要被重点平衡。
- 终结卡密度应形成自然取舍：前期 1-2 张更稳定，后期只有在过牌、减费、回费、复制或连锁保留能力足够时才适合加入更多终结卡。
- 多终结卡流派可以存在，且终结卡之间可以联动，但需要承担更高构筑成本和卡手风险。
- 终结卡数值可以采用固定值、随连锁层数加算或随连锁层数乘算；乘算收益必须有更高阈值、资源成本或使用限制。
- 秒杀非 Boss 敌人的终结卡可以存在，但必须排除 Boss，并通过阈值、目标范围、稀有度或触发次数限制控制强度。
- 基础平衡可优先关注单体高伤害、群体伤害、暴击率提升、抽牌 / 回费等候选终结卡，验证核心手感和数值边界。
- 连锁阈值基准采用 3 / 5 / 8 三档：3 层负责教学反馈，5 层负责续航连锁，8 层负责成型构筑的高爆发。
- 连锁层数不设置硬上限；超过 8 层后的收益应重点控制，避免无限堆叠压倒其他决策。
- 连锁层数机制应奖励高手规划，但不能让未成型卡组完全失去基础战斗能力。
- 随机性不应长期掩盖玩家决策影响。

## 平衡取舍边界

早期平衡目标是避免无解、误导和单一最优，而不是过早追求所有流派完全均等。若某个强 combo 能显著提升爽感和分享欲，但仍有构筑成本、出牌条件或风险，可以优先保留并观察。

## 平衡对象

- 卡牌费用、效果和升级。
- 终结卡密度、抽到终结卡的稳定性和卡手概率。
- 连锁层数增长、阈值、超过 8 层后的收益曲线、清空规则、保留规则、固定 / 加算 / 乘算收益和终结卡效果类型。
- 单体高伤害、群体伤害、暴击率提升、抽牌 / 回费等候选效果的收益边界。
- 3 / 5 / 8 三档阈值的触发频率、收益差距和构筑要求。
- 敌人生命、伤害和行为。
- 遗物触发频率和收益。
- 奖励池出现概率。
- 路线节点密度。
- 商店价格。
- 治疗和资源恢复。

## 调整优先级

1. 修复无解、误导或不可读问题。
2. 削弱无条件最优策略。
3. 增强有趣但缺少支撑的构筑。
4. 微调数值效率。
