# 指标与测试数据

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义试玩时需要记录和观察的数据，用于平衡彩能、色彩、武器卡池、魔物和奖励。

## 第一版 MVP 必记录指标

- Run 种子。
- 起始武器选牌：本局主手 / 副手选择结果、主手武器选择的 6 张、副手武器选择的 4 张。
- 节点顺序。
- 每场战斗回合数、受伤、死亡或胜利。
- 每回合生成彩能数量。
- 每回合彩能颜色构成。
- 回合结束被清空的未使用彩能。
- 终结牌使用次数、消耗彩能数量和颜色构成。
- 终结牌色彩附加效果触发次数。
- 色彩碎片获得颜色、附魔对象和附魔后使用次数。
- 战后卡牌三选一候选、选择结果和所属武器。
- 遗物获得节点和触发次数。
- Boss 简化奖励类型和领取结果。
- MVP 通关或失败节点。
- 总时长。

## 观察问题

- 玩家是否理解彩能跨回合不保留。
- 玩家是否能预测行动牌会生成什么颜色的彩能。
- 玩家是否能理解终结牌消耗彩能后的色彩附加效果。
- 玩家是否因为起始武器选牌负担过高而卡住。
- 玩家是否觉得每战附魔太快、太慢或太随机。
- 5 种色彩是否都有选择理由。
- Boss 奖励是否让玩家期待下一阶段。
