# 指标与测试数据

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义试玩时需要记录和观察的数据，用于平衡拍位编排、动作对撞、无色彩能、武器卡池、魔物和奖励。五色色彩与附魔指标留到后续阶段。

## 第一版 MVP 必记录指标

- Run 种子。
- 起始武器选择：本局主手 / 副手选择结果、系统自动加入的主手 6 张和副手 4 张行动牌。
- 节点顺序。
- 每场战斗轮数、受伤、死亡或胜利。
- 每轮生成无色彩能数量。
- 轮末被清空的未使用彩能。
- 终结牌使用次数和消耗彩能数量。
- 行动牌放入拍位后的取消次数、无效目标次数和成功锁定目标次数。
- 对撞魔物拍位次数与空门直击次数。
- 格挡有效次数、闪避成功次数和成功伤害次数。
- 战后卡牌三选一候选、选择结果和所属武器。
- 遗物获得节点和触发次数。
- Boss 简化奖励类型和领取结果。
- MVP 通关或失败节点。
- 总时长。

## 观察问题

- 玩家是否理解轮末未使用彩能不保留。
- 玩家是否能预测哪些动作成功后会生成彩能。
- 玩家是否能理解终结牌消耗无色彩能后的收益。
- 玩家是否因为主副武器选择说明不清而卡住；移除起始牌自由选择后，开局负担是否明显下降。
- 玩家是否因为放入拍位后立即指向目标而误操作。
- 玩家是否能理解对撞目标与空门目标的区别。
- Boss 奖励是否让玩家期待下一阶段。
