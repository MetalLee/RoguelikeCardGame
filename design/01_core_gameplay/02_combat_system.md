# 战斗系统

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义卡牌战斗的基本规则、回合结构、敌人意图、伤害防御和状态系统。

## 战斗回合结构

1. 回合开始：抽牌、获得资源、触发开始时效果。
2. 玩家行动：打出卡牌、选择目标、触发组合。
3. 回合结束：弃牌、结算持续效果、敌人行动。
4. 新回合：敌人意图更新，玩家重新评估。

## 基础要素

- 生命：战斗和路线资源，承担长期风险。
- 能量或行动资源：限制每回合行动密度。
- 连锁层数：玩家打出卡牌时积累的回合内战斗节奏资源，用于触发终结卡在高连锁条件下的额外强力效果。
- 抽牌与弃牌：决定构筑稳定性和循环速度。
- 防御：短期伤害管理。
- 状态：提供跨回合策略和敌我机制差异。
- 敌人意图：让玩家提前理解主要威胁。

## 连锁层数与 Combo 爽感

连锁层数是当前确认的核心战斗机制之一。玩家通过连续打出卡牌提升连锁层数，并通过牌序规划把层数推到关键阈值，再使用带有高连锁额外效果的终结卡兑现爆发。

默认生命周期：

- 连锁层数是回合内资源，通常在回合结束时归零。
- 使用终结卡后，当前连锁层数通常归零。
- 某些遗物或特殊终结卡可以保留部分或全部连锁层数，作为突破默认节奏的高价值构筑点。
- 保留效果必须明确写出保留比例、保留数量、触发时机和是否有次数限制。

连锁阈值基准：

- 3 层：低阈值，用于让玩家理解连锁机制，并快速看到终结卡被增强的反馈。
- 5 层：中阈值，用于触发续航连锁效果，例如抽牌、回费或继续延展出牌链条。
- 8 层：高阈值，用于卡组成型后的强爆发，例如巨额群体伤害；秒杀非 Boss 敌人可作为同档位的高冲击扩展效果。
- 连锁层数不设置硬上限；超过 8 层后仍可继续计数，具体收益曲线和 UI 表达后续定义。

设计目标：

- 让玩家在出牌顺序中产生明确规划，例如先用低费牌、抽牌牌或铺垫牌抬高连锁，再打出终结卡。
- 高连锁效果应明显强于基础效果，形成巨额伤害、强控制、资源返还或其他足够醒目的战斗收益。
- 终结卡效果可以包括伤害增加、暴击率增加、群体伤害、保留连锁、抽牌、赋予敌人异常状态、赋予自身增益，以及秒杀非 Boss 敌人等特殊效果。
- 秒杀类效果必须排除 Boss，并明确目标限制、触发阈值和可读反馈。
- 连锁爆发必须有清楚的 UI 和动画反馈，让玩家理解“为什么这张牌现在变强了”。
- 高连锁终结卡需要有足够打击感：命中节奏、停顿、伤害数字、屏幕反馈、音效和动画应共同放大兑现瞬间。
- 连锁机制应提供爽感，但不能让所有回合都退化为无脑堆层数；敌人压力、资源限制、手牌结构和防御需求应持续制造取舍。

## 设计准则

- 敌人意图应清楚表达即将发生的伤害、强化、削弱或特殊行动。
- 玩家每回合应在进攻、防御、铺垫、资源保留之间做取舍。
- 玩家每回合应评估是否继续抬高连锁层数，还是提前用终结卡兑现收益。
- 状态效果命名和图标必须稳定，避免同义机制重复。
- 难点应来自可理解的压力组合，而不是隐藏规则。

## 待定义系统

- 是否允许保留手牌。
- 是否存在行动点以外的第二资源。
- 超过 8 层后的收益曲线、UI 表达和防失控约束。
- 保留连锁层数的遗物和特殊终结卡应如何定价与限制。
- 防御是否回合末清空。
- 状态层数和持续时间规则。
