# 战斗系统

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义卡牌战斗的基本规则、回合结构、敌人意图、伤害防御、彩能、色彩和状态系统。

## 战斗回合结构

1. 回合开始：清空上一轮敌人行动后剩余的防御，抽 5 张牌，按当前行动点上限 + 额外行动点数恢复行动点，触发开始时效果。
2. 玩家行动：打出行动牌或终结牌、选择目标、生成彩能、触发色彩或遗物效果。
3. 玩家回合结束：未使用行动点清空，未使用手牌全部弃掉，未使用彩能清空，结算回合结束效果。
4. 敌人行动：按敌人意图结算伤害、强化、削弱、控制或特殊行动；当前防御用于抵挡敌人伤害。
5. 新回合准备：敌人意图更新，玩家重新评估。

## 基础要素

- 生命：战斗和路线资源。第一版 MVP 每场战斗开始时生命回满，用于稳定测试单场战斗；正式版本方向为跨战斗保留且不自动恢复，承担长期风险。
- 行动点：限制每回合行动牌使用密度；基础上限为 3，每回合恢复当前行动点上限 + 额外行动点数。
- 彩能：原“连锁层数”的世界观命名和新版规则表达，是回合内战斗节奏资源。行动牌主要生成彩能，终结牌消耗彩能。基础彩能上限为 6，跨回合不保留。
- 色彩：附着在彩能上的属性。行动牌被色彩碎片附魔后，后续生成的彩能携带对应色彩；终结牌消耗彩能时，同时结算被消耗彩能携带的色彩附加效果。
- 抽牌与弃牌：默认每回合抽 5 张牌；抽牌堆不足时，将弃牌堆洗回抽牌堆继续抽，决定构筑稳定性和循环速度。
- 防御：短期伤害管理；玩家回合结束时不清空，敌人行动后若仍有剩余，则在下一回合开始时清空。
- 状态：提供跨回合策略和敌我机制差异。
- 敌人意图：让玩家提前理解主要威胁。

## 卡牌类型

卡牌类型收束为两类：

- 行动牌：主要承担攻击、防御、控制、过牌、回费、减费和生成彩能等职责。行动牌通常消耗行动点，但可以存在 0 费或特殊费用行动牌。行动牌默认不再用“每张 +1 连锁”的抽象表述，而应在卡牌效果中明确写出生成多少点彩能，以及这些彩能是否继承行动牌附魔色彩。
- 终结牌：消耗彩能造成强大的伤害或效果，并根据被消耗彩能的数量与色彩追加结算。终结牌默认不消耗行动点，但必须满足自身标注的彩能需求或消耗规则。

不再设置独立“技能牌”类型。原本防御、抽牌、检索、减费、回费、状态、控制或牌序调整等辅助职责，统一进入行动牌或终结牌。

## 彩能与终结牌

彩能是当前确认的核心战斗机制之一。玩家通过行动牌和少数特殊效果获得彩能，并通过牌序规划在同一回合内积累足够彩能，再使用终结牌兑现爆发。

默认生命周期：

- 彩能是回合内资源，通常在玩家回合结束时归零。
- 彩能基础上限为 6；若卡牌、遗物或色彩核心允许突破上限，必须在文本中明确写出。
- 终结牌需要满足自身彩能需求后才能打出。
- 终结牌可以消耗固定数量彩能、X 点彩能或全部当前彩能，具体由卡牌文本决定。
- 使用终结牌后，被消耗的彩能移除；未消耗彩能是否保留到本回合后续，必须由终结牌文本明确。
- 不再使用 3 / 5 / 8 固定阈值；终结牌的彩能条件、消耗数量和随数量增强方式由单卡定义。

设计目标：

- 让玩家在出牌顺序中产生明确规划，例如先用左轮剑斩击生成彩能，再用机械臂防护或控制稳定局势，最后用终结牌消耗彩能。
- 终结牌效果应明显强于普通行动牌，形成巨额伤害、强控制、资源返还、高额防御、回复或其他醒目的战斗收益。
- 终结牌默认不消耗行动点，因此其主要限制应来自彩能数量、彩能颜色、手牌结构、抽到时机和是否值得提前兑现。
- 高彩能终结牌需要有足够打击感：命中节奏、停顿、伤害数字、屏幕反馈、音效和动画应共同放大兑现瞬间。

## 色彩机制

初版设置 5 种色彩：

- 红色：力量。附加伤害效果，适合输出和爆发终结牌。
- 黄色：速度。增加卡牌释放次数、追加段数或复制轻量效果，适合多段和行动延展。
- 蓝色：防卫。附加护盾 / 防御效果，适合抗压和反击。
- 绿色：生命。附加回复、生命转化或续航效果，适合长线稳定。
- 紫色：神秘。附加翻倍、放大或异常效果，适合高风险高收益。

色彩通过色彩碎片附魔到行动牌上。行动牌产生的彩能携带该行动牌的色彩；未附魔行动牌产生无色彩能。终结牌消耗彩能时，根据所有被消耗彩能的色彩逐个或汇总触发附加效果，具体叠加规则进入卡牌模板和数据管线阶段细化。

## 默认回合结束结算

- 未使用行动点在玩家回合结束时清空。
- 未使用彩能在玩家回合结束时清空。
- 未使用手牌在玩家回合结束时全部弃掉。
- 防御在玩家回合结束时不清空，用于承受随后敌人行动造成的伤害。
- 敌人行动后仍剩余的防御，会在下一回合开始时清空。
- 下一回合开始时抽 5 张牌，并按当前行动点上限 + 额外行动点数恢复行动点；无修正时恢复 3 点。
- 少数行动牌、终结牌或遗物可以突破默认规则，保留手牌、防御、行动点或部分彩能。

## 默认牌库循环

- 抽牌堆不足以满足抽牌需求时，将弃牌堆洗回抽牌堆继续抽。
- 普通卡牌在同一场战斗中可以多轮循环使用；能否再次打出仍受行动点、手牌、彩能、色彩、目标规则和其他卡牌条件限制。
- 过牌能力会加快卡组循环，让玩家更频繁地找到彩能生成牌、附魔行动牌和终结牌。
- 部分卡牌效果可以强制将弃牌堆整体洗回抽牌堆，用于制造检索、循环或爆发回合。
- 消耗牌、临时牌和特殊牌区留到后续卡牌模板阶段定义。

## 第一版 MVP 战斗反馈资源

当前已有 VFX、卡牌模板和 UI 资源大多围绕旧“连锁层数 / 技能牌 / 3/5/8 阈值”设计。它们可作为历史美术参考，但需要在后续资源迭代中迁移为“彩能 / 色彩 / 双卡牌类型”表达。

表现层必须继续遵守原则：VFX 只表达反馈，不承担规则判断；伤害、防御、彩能变化、色彩结算和终结牌效果仍以规则层结算与结构化日志为准。

## 设计准则

- 敌人意图应清楚表达即将发生的伤害、强化、削弱或特殊行动。
- 玩家每回合应在进攻、防御、铺垫彩能、保留行动点和提前终结之间做取舍。
- 玩家每回合应评估是否继续积累彩能，还是提前用终结牌兑现收益。
- 色彩效果命名和图标必须稳定，避免同义机制重复。
- 难点应来自可理解的压力组合，而不是隐藏规则。

## 待定义系统

- 彩能上限是否永远为 6，还是可被遗物、武器或色彩核心突破。
- 无色彩能参与终结牌时的默认规则。
- 多个相同色彩被消耗时，是逐点触发、按数量加算，还是触发一次并按数量增强。
- 黄色追加释放和紫色翻倍的防失控边界。
- 色彩核心如何提升附魔效果。
- 保留彩能、改变彩能颜色、复制彩能或转换彩能的价值边界。
- 状态层数和持续时间规则。
