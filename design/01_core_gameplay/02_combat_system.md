# 战斗系统

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义卡牌战斗的基本规则、回合结构、敌人意图、伤害防御和状态系统。

## 战斗回合结构

1. 回合开始：抽牌、获得资源、触发开始时效果。
2. 玩家行动：打出卡牌、选择目标、触发组合。
3. 回合结束：弃牌、结算持续效果、敌人行动。
4. 新回合：敌人意图更新，玩家重新评估。

## 基础要素

- 生命：战斗和路线资源，承担长期风险。
- 能量或行动资源：限制每回合行动密度。
- 抽牌与弃牌：决定构筑稳定性和循环速度。
- 防御：短期伤害管理。
- 状态：提供跨回合策略和敌我机制差异。
- 敌人意图：让玩家提前理解主要威胁。

## 设计准则

- 敌人意图应清楚表达即将发生的伤害、强化、削弱或特殊行动。
- 玩家每回合应在进攻、防御、铺垫、资源保留之间做取舍。
- 状态效果命名和图标必须稳定，避免同义机制重复。
- 难点应来自可理解的压力组合，而不是隐藏规则。

## 待定义系统

- 是否允许保留手牌。
- 是否存在行动点以外的第二资源。
- 防御是否回合末清空。
- 状态层数和持续时间规则。
