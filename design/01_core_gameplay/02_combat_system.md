# 战斗系统

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义卡牌战斗的基本规则、回合结构、敌人意图、伤害防御和状态系统。

## 战斗回合结构

1. 回合开始：清空上一轮敌人行动后剩余的防御，抽 5 张牌，按当前行动点上限 + 额外行动点数恢复行动点，触发开始时效果。
2. 玩家行动：打出卡牌、选择目标、触发组合。
3. 玩家回合结束：未使用行动点清空，连锁层数清空，未使用手牌全部弃掉，结算回合结束效果。
4. 敌人行动：按敌人意图结算伤害、强化、削弱或特殊行动；当前防御用于抵挡敌人伤害。
5. 新回合准备：敌人意图更新，玩家重新评估。

## 基础要素

- 生命：战斗和路线资源。第一版 MVP 每场战斗开始时生命回满，用于稳定测试单场战斗；正式版本方向为跨战斗保留且不自动恢复，承担长期风险。
- 行动点：限制每回合行动牌使用密度；基础上限为 3，每回合恢复当前行动点上限 + 额外行动点数，行动牌默认消耗行动点，技能牌和终结牌默认不消耗行动点。
- 连锁层数：玩家通过行动牌等手段积累的回合内战斗节奏资源，用于触发终结牌在高连锁条件下的额外强力效果。
- 抽牌与弃牌：默认每回合抽 5 张牌；抽牌堆不足时，将弃牌堆洗回抽牌堆继续抽，决定构筑稳定性和循环速度。
- 防御：短期伤害管理；玩家回合结束时不清空，敌人行动后若仍有剩余，则在下一回合开始时清空。
- 状态：提供跨回合策略和敌我机制差异。
- 敌人意图：让玩家提前理解主要威胁。

## 连锁层数与 Combo 爽感

连锁层数是当前确认的核心战斗机制之一。玩家通过行动牌和少数特殊效果提升连锁层数，并通过牌序规划把层数推到关键阈值，再使用带有高连锁额外效果的终结牌兑现爆发。

卡牌类型与默认连锁规则：

- 行动牌：主要负责攻击和叠连锁层数；默认消耗行动点，使用后 +1 连锁层数，是连锁增长的主要来源。行动牌费用与连锁增长分离，不论费用高低，默认每打出 1 张行动牌只增加 1 层连锁。
- 技能牌：默认不消耗行动点，默认不产生连锁层数，主要承担防御、抽牌、检索、减费、回费、状态、控制或牌序调整等职责，是辅助和稳定工具。
- 终结牌：默认消耗所有当前连锁层数，且默认不消耗行动点；需要满足自身连锁层数使用条件后才能打出，用于兑现高连锁收益。
- 少数卡牌和遗物可以突破默认规则，提供额外增加层数、额外消耗层数、保留层数或其他特殊连锁结算。
- 特殊效果必须在卡牌或遗物文本中明确写出触发时机、层数变化、是否消耗连锁，以及是否覆盖默认规则。

默认生命周期：

- 连锁层数是回合内资源，通常在玩家回合结束时归零。
- 终结牌需要满足自身连锁层数使用条件后才能打出；使用终结牌后，当前连锁层数通常归零。
- 某些遗物或特殊终结牌可以保留部分或全部连锁层数，作为突破默认节奏的高价值构筑点。
- 保留效果必须明确写出保留比例、保留数量、触发时机和是否有次数限制。

默认回合结束结算：

- 未使用行动点在玩家回合结束时清空。
- 连锁层数在玩家回合结束时清空。
- 未使用手牌在玩家回合结束时全部弃掉。
- 防御在玩家回合结束时不清空，用于承受随后敌人行动造成的伤害。
- 敌人行动后仍剩余的防御，会在下一回合开始时清空。
- 下一回合开始时抽 5 张牌，并按当前行动点上限 + 额外行动点数恢复行动点；无修正时恢复 3 点。
- 少数技能牌、终结牌或遗物可以突破默认规则，保留手牌、防御、行动点或连锁层数。

默认牌库循环：

- 抽牌堆不足以满足抽牌需求时，将弃牌堆洗回抽牌堆继续抽。
- 普通卡牌在同一场战斗中可以多轮循环使用；能否再次打出仍受行动点、手牌、连锁层数、终结牌使用条件和其他卡牌条件限制。
- 过牌能力会加快卡组循环，让玩家更频繁地找到启动牌、延展牌和终结牌，是高连锁构筑的重要基础设施。
- 部分卡牌效果可以强制将弃牌堆整体洗回抽牌堆，用于制造检索、循环或爆发回合。
- 消耗牌、临时牌和特殊牌区留到后续卡牌模板阶段定义。

连锁阈值基准：

- 3 层：低阈值，用于让玩家理解连锁机制，并快速看到终结牌被增强的反馈。
- 5 层：中阈值，用于触发续航连锁效果，例如抽牌、回费或继续延展出牌链条。
- 8 层：高阈值，用于卡组成型后的强爆发，例如巨额群体伤害；秒杀非 Boss 敌人可作为同档位的高冲击扩展效果。
- 连锁层数不设置硬上限；超过 8 层后仍可继续计数，具体收益曲线和 UI 表达后续定义。

设计目标：

- 让玩家在出牌顺序中产生明确规划，例如先用低费行动牌、抽牌牌或铺垫牌抬高连锁，再打出终结牌。
- 高连锁效果应明显强于基础效果，形成巨额伤害、强控制、资源返还或其他足够醒目的战斗收益。
- 终结牌效果可以包括爆发伤害、高额防御、伤害增加、暴击率增加、群体伤害、保留连锁、抽牌、回费、赋予敌人异常状态、赋予自身增益，以及秒杀非 Boss 敌人等特殊效果。
- 终结牌默认不消耗行动点，因此其主要限制应来自连锁层数、手牌结构、抽到时机和是否值得提前兑现。
- 初始牌组中的终结牌至少需要 3 层连锁才能使用，定位为单体爆发型终结牌：3 层造成高额单体伤害，5 层连锁及以上造成翻倍伤害，用于教学终结牌使用条件和连锁层数对效果强度的影响。
- 初始终结牌不承担防御、抽牌、回费、异常状态、群体伤害、保留连锁等复杂功能；这些特殊终结牌效果应通过闯关奖励、构筑转向或流派扩展逐步出现。
- 初始牌组中的 6 张行动牌采用强弱对比结构：4 张 1 费普通攻击、1 张 0 费弱攻击、1 张 2 费强攻击。该结构用于教学费用影响伤害、回合节奏和行动点机会成本，但不影响默认连锁增长规则。
- 0 费行动牌需要控制基础强度，避免无成本堆连锁过于稳定；2 费行动牌可通过更高伤害或更强效果体现成本，但默认仍只 +1 连锁。
- 初始牌组中的 2 张基础防御技能牌提供防御值并 +1 行动点，用于让玩家在防守回合仍能延展行动牌链条。
- 初始牌组中的 1 张过牌技能牌抽 2 张牌，用于提高玩家找到行动牌或终结牌的概率。
- 初始技能牌可以辅助玩家追到 3 层或 5 层连锁，但默认仍不增加连锁层数。
- 初始卡组采用保守数值风格，方便后续通过奖励感到成长明显；具体伤害值、防御值和终结牌基础伤害留到最小原型阶段统一设定和测试。
- 基础行动点上限为 3；遗物或技能可以提高行动点上限，从而提高每回合基础恢复量。
- 行动点上限不作为当前行动点不可超过的绝对硬上限；卡牌、遗物或其他效果可以继续增加当前回合行动点，用于支持更长行动牌链条和更高连锁层数。
- 某些卡牌可以预先增加下回合的额外行动点数；每回合恢复的行动点数 = 当前行动点上限 + 当前回合应获得的额外行动点数。
- 回费、减费、临时行动点增加、行动点上限提升和下回合额外行动点需要在效果文本中明确区分。
- 默认每回合抽 5 张牌；牌组构筑完善后，技能牌和终结牌可以通过过牌提高卡组运转能力。
- 抽牌堆不足时会将弃牌堆洗回抽牌堆继续抽；只要行动点、连锁层数和其他使用条件满足，普通卡牌可以在同一场战斗中多轮循环使用。
- 秒杀类效果必须排除 Boss，并明确目标限制、触发阈值和可读反馈。
- 连锁爆发必须有清楚的 UI 和动画反馈，让玩家理解“为什么这张牌现在变强了”。
- 高连锁终结牌需要有足够打击感：命中节奏、停顿、伤害数字、屏幕反馈、音效和动画应共同放大兑现瞬间。
- 连锁机制应提供爽感，但不能让所有回合都退化为无脑堆层数；敌人压力、资源限制、手牌结构和防御需求应持续制造取舍。

## 设计准则

- 敌人意图应清楚表达即将发生的伤害、强化、削弱或特殊行动。
- 玩家每回合应在进攻、防御、铺垫、资源保留之间做取舍。
- 玩家每回合应评估是否继续抬高连锁层数，还是提前用终结牌兑现收益。
- 状态效果命名和图标必须稳定，避免同义机制重复。
- 难点应来自可理解的压力组合，而不是隐藏规则。

## 待定义系统

- 行动点上限提升、当前回合行动点增加和下回合额外行动点的数值边界与 UI 表达。
- 普通重洗和强制重洗的 UI、动画与文本表达。
- 是否存在行动点以外的第二战斗资源。
- 超过 8 层后的收益曲线、UI 表达和防失控约束。
- 保留连锁层数的遗物和特殊终结牌应如何定价与限制。
- 保留手牌、防御、行动点或连锁层数的例外效果应如何定价与限制。
- 状态层数和持续时间规则。
