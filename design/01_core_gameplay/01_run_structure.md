# 单局结构

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义一局游戏的阶段结构、节点类型、节奏曲线和失败条件。

## 标准 Run 结构

- 起始选择：主角、主手武器、副手武器和路线预览。
- 早期：教学式压力，鼓励理解三拍拆招、魔物拍位、成功产彩和终结槽释放。
- 中期：魔物机制开始针对构筑短板，奖励提供转向或强化。
- 后期：资源压力增加，路线选择更高风险。
- 终局：Boss 或特殊挑战检验武器卡组、拍位编排、彩能生产和终结牌使用时机；色彩搭配进入后续阶段。

## 时长目标

完整版本的单局目标时长为 30-40 分钟。Run 结构应服务中等时长逐步成型：早期让玩家识别武器和三拍拆招方向，中期形成可运行联动，后期提供彩能 / 终结牌高光；色彩附魔在后续阶段加入后再承担长期构筑分化。

## 第一版 MVP Run 范围

第一版 MVP 采用 1 个线性章节，目标时长约 10-15 分钟，主要验证战斗局内循环。战斗顺序为：普通战斗 1 -> 普通战斗 2 -> 普通战斗 3 -> 精英战斗 -> 普通战斗 4 -> Boss 战斗。6 场战斗全部采用固定遭遇，等战斗循环成立后再加入普通战斗随机池。

第一版 MVP 暂不做路线图、商店、休息或完整货币经济。事件系统是下一阶段优先方向，但当前 MVP 先打磨三拍战斗系统、无色彩能终结和最小奖励闭环。

第一版 MVP 起始角色固定为「左」，但起始武器顺序不固定。玩家需要在左轮剑和机械臂之间自由选择主手与副手；进入 Run 时，系统自动把主手武器固定初始池的 6 张卡全部加入初始卡组，并把副手武器固定初始池中的 4 张行动牌加入初始卡组，组成 10 张。MVP 保留主副武器选择界面，但不再提供起始卡牌自由选择界面。

第一版 MVP 每场战斗开始时玩家生命回满，优先验证单场三拍战斗、固定遭遇、无色彩能与奖励闭环。正式版本方向为生命跨战斗保留且不自动恢复；本轮不展开正式版本恢复节点或恢复经济。

第一版 MVP 中 Boss 击败即显示 MVP 通关并提供重开入口，同时提供简化 Boss 奖励反馈。后续完整 Run 的阶段性 Boss 可继续承担构筑转折点奖励职责。

前 3 场普通战斗用于建立基础规则理解并提供武器卡牌奖励；精英战斗用于暴露拆招与产彩短板并给予普通遗物；精英后的普通战斗用于让玩家感受遗物和新卡影响，并在 Boss 前获得最后一次调整机会；Boss 战斗用于综合检验构筑和战斗判断。色彩附魔奖励在后续阶段恢复。

第一版 MVP 的六场固定战斗采用教学递进型职责分配：

- 普通战斗 1：教学拖行动牌入拍、自动瞄准魔物拍和成功产无色彩能。
- 普通战斗 2：教学格挡 / 闪避动作、防护职责和魔物拍位威胁读取。
- 普通战斗 3：教学多魔物、魔物拍锁定和空门本体目标。
- 精英战斗：检验攻防节奏、拍位目标分配和无色彩能生成稳定性。
- 普通战斗 4：让玩家试用遗物、新卡和终结牌节奏。
- Boss 战斗：综合检验拍位编排、彩能积累、终结牌、防御和奖励构筑成果。

## 节点类型

- 普通战斗：当前切片优先提供武器卡牌三选一；色彩碎片附魔延后到五色阶段。后续恢复时，每场普通战斗可随机给 1 个色彩碎片，并提供 3 张左轮剑 / 机械臂相关卡牌候选。
- 精英战斗：高风险高回报，检验当前强度；除普通战斗奖励外，额外给 1 个普通遗物。第一版 MVP 可先使用固定普通遗物。
- Boss：阶段性规则考试；第一版 MVP 的终点 Boss 击败后直接胜利，并给出简化 Boss 奖励反馈。后续完整 Run 中 Boss 可奖励 Boss 遗物、Boss 特有卡牌和色彩核心。
- 事件：提供信息不完全的风险回报选择，并承担碎片化叙事。事件不进入当前 MVP，但下一阶段优先级高于商店和路线图。
- 商店：资源转换和定向补强，下一阶段不优先。
- 休息或锻造：恢复与强化之间的取舍，后续阶段定义。
- 宝箱或特殊奖励：节奏缓冲与惊喜，后续阶段定义。

## 失败条件

- 生命归零。
- 第一版 MVP 中，生命归零立即结束本次 MVP Run，回到开局或失败结算；玩家流程不提供当前战斗重试。
- 特殊模式或挑战条件失败，待定。

## 设计准则

- 每个阶段都应有不同的压力主题。
- 普通战斗奖励应帮助玩家定向补强武器卡组；色彩附魔路线在后续阶段恢复后再进入普通奖励核心。
- 精英和 Boss 不只是更高数值；精英通过普通遗物提供机制差异，Boss 通过简化 Boss 奖励预告正式版本的阶段性构筑转折。
- 完整版本的路线图应让玩家能做战略规划，但保留局部未知；第一版 MVP 暂不做路线图。
- 完整版本的构筑成型和终结牌高光应在 30-40 分钟目标时长内自然出现；第一版 MVP 只验证最小构筑成长和终结牌兑现是否成立。
