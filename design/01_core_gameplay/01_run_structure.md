# 单局结构

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义一局游戏的阶段结构、节点类型、节奏曲线和失败条件。

## 标准 Run 结构

- 起始选择：角色、初始牌组、初始奖励或路线预览。
- 早期：教学式压力，鼓励建立初步构筑方向。
- 中期：敌人机制开始针对构筑短板，奖励提供转向或强化。
- 后期：资源压力增加，路线选择更高风险。
- 终局：Boss 或特殊挑战检验构筑完整度与操作理解。

## 节点类型

- 普通战斗：提供基础奖励和构筑调整机会。
- 精英战斗：高风险高回报，检验当前强度。
- Boss：阶段性规则考试。
- 事件：提供信息不完全的风险回报选择。
- 商店：资源转换和定向补强。
- 休息或锻造：恢复与强化之间的取舍。
- 宝箱或特殊奖励：节奏缓冲与惊喜。

## 失败条件

- 生命归零。
- 特殊模式或挑战条件失败，待定。

## 设计准则

- 每个阶段都应有不同的压力主题。
- 精英和 Boss 不只是更高数值，应提供机制差异。
- 路线图应让玩家能做战略规划，但保留局部未知。
