# 资源与经济

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义局内资源、战斗资源、商店经济和资源转换关系。

## 资源类型

- 生命：正式版本中的跨战斗延续风险资源，方向为跨战斗保留且不自动恢复；第一版 MVP 每场战斗开始时生命回满，用于优先验证单场战斗和奖励闭环，不展开恢复节点或恢复经济。
- 行动点：限制每回合行动牌使用密度。行动牌默认消耗行动点，技能牌和终结牌默认不消耗行动点；基础行动点上限为 3，每回合恢复的行动点数为当前行动点上限 + 额外行动点数，未使用行动点在玩家回合结束时清空。
- 货币：用于商店、事件、删除或特殊服务。
- 卡牌奖励：构筑扩张资源，也可能带来牌库污染。普通战斗后从行动牌包、技能牌包、终结牌包中选择 1 包打开，选择前只知道卡牌包类型，不知道具体卡牌，并从打开后的 3 张同类型候选卡牌中选择 0-3 张加入卡组。
- 过牌能力：默认每回合抽 5 张牌；抽牌堆不足时，将弃牌堆洗回抽牌堆继续抽。构筑完善后，技能牌和终结牌可通过过牌提高卡组运转能力，影响玩家寻找启动牌、延展牌和终结牌的稳定性，也决定卡组能否承载更高终结牌密度。
- 遗物或被动：长期规则改变资源。
- 信息：路线、敌人、事件结果的可预判程度。

## 经济设计准则

- 资源应形成可比较的转换关系，例如生命换奖励、货币换定向补强。
- 商店价格应让玩家无法买下所有想要的东西。
- 事件代价应明确，隐藏结果应控制在可接受范围。
- 资源稀缺应制造策略，而不是让玩家频繁陷入无解局面。
- 普通战斗卡牌奖励通过“选择卡牌包类型 + 选择 0-3 张加入”的两层决策控制构筑方向和卡组厚度。
- 选择卡牌包前只显示类型，不预览具体卡牌；打开后的候选牌若不适合当前构筑，玩家可以选择 0 张来避免牌组污染。
- 第一版 MVP 的卡牌包采用固定可重复池：每次打开某类型卡牌包，都显示该类型固定 3 张奖励牌；拿过的牌仍可再次出现，允许重复加入同名牌；不做奖励牌移除、去重或候选不足补位。
- 每个卡牌包至少包含 1 张高于普通稀有度的卡牌，保证开包后有基础质量亮点。
- 早期卡牌包以普通牌为主，中后期逐步提高高稀有卡牌出现概率，让奖励质量随 Run 推进成长。
- 选择 0 张是控制牌组污染的基础手段；选择多张则是在奖励高度匹配时加速构筑成型的手段。
- 少拿或不拿卡牌默认没有额外补偿；控制卡组厚度本身就是该选择的收益。
- 跳过收益、少拿收益或拒绝卡牌收益留到商店、货币和删牌经济确定后再评估，避免普通战斗奖励过早变成稳定货币转换入口。
- 行动牌包、技能牌包和终结牌包应分别服务不同构筑需求：补充连锁增长、补充防御 / 资源调度、补充连锁兑现点。
- 精英战斗在卡牌包之外额外随机给 1 个普通遗物，形成高风险节点的额外规则收益；第一版 MVP 只有 1 个普通遗物，因此该奖励等同固定获得。
- Boss 战斗给稀有度更高的卡牌包，并额外提供 3 个能够影响构筑、改变玩法的特殊遗物候选，玩家选择 1 个获得，作为阶段性构筑转折点。
- 终结牌、过牌、重洗、减费、回费、临时行动点增加、复制和连锁保留之间应形成构筑经济：终结牌提供爆发，其他资源决定终结牌能否稳定启动、循环和连续兑现。
- 资源型终结牌可优先通过抽牌 / 回费验证，观察它们能否支撑更长出牌链条和更多终结牌密度。
- 高终结牌密度必须依赖足够过牌或检索能力，否则应自然产生卡手风险。
- 抽牌堆不足时会将弃牌堆洗回抽牌堆继续抽；普通卡牌只要行动点、连锁层数和其他使用条件满足，就可以在同一场战斗中多轮循环使用。
- 部分卡牌效果可以强制将弃牌堆整体洗回抽牌堆，用于提高关键牌回流速度；这类效果需要通过抽牌量、检索范围、牌序信息和敌人压力控制强度。
- 行动点主要约束行动牌的攻击与叠连锁节奏；技能牌和终结牌默认不消耗行动点，因此更依赖功能定位、手牌结构、连锁层数使用条件和抽牌稳定性作为限制。
- 基础行动点上限为 3。遗物或技能可以提高行动点上限；行动点上限决定每回合基础恢复量，而不是当前行动点不可超过的绝对硬上限。
- 当前回合行动点可以被卡牌、遗物或其他效果继续增加，用于支持更长行动牌链条和更高连锁层数；默认玩家回合结束时未使用行动点仍然清空。
- 某些卡牌可以预先增加下回合的额外行动点数。每回合恢复的行动点数 = 当前行动点上限 + 当前回合应获得的额外行动点数。
- 卡牌文本需要区分三类行动点效果：当前回合获得行动点、提高行动点上限、增加下回合额外行动点。
- 行动牌费用与连锁增长分离。行动牌费用可高可低，但默认每打出 1 张行动牌只增加 1 层连锁；费用主要影响伤害、回合节奏和行动点机会成本。
- 技能牌、终结牌和遗物可以提供回费、减费、临时行动点增加、行动点上限提升或下回合额外行动点，用于支持更长行动牌链条和更高连锁层数。
- 技能牌和终结牌可以通过过牌提高卡组运转能力，帮助构筑成型后更稳定地延展行动牌链条和寻找终结牌。
- 默认回合结算会清空未使用行动点、连锁层数和未使用手牌。防御不会在玩家回合结束时清空，而是在敌人行动后、下一回合开始时清空剩余值。
- 少数技能牌、终结牌或遗物可以保留手牌、防御、行动点或连锁层数，作为突破默认资源结算的高价值构筑点。
- 初始牌组基准为 10 张牌：4 张 1 费普通攻击、1 张 0 费弱攻击、1 张 2 费强攻击、2 张基础防御技能牌、1 张过牌技能牌、1 张终结牌。该结构提供稳定的行动点消耗对象、基础防御、最低过牌能力和一次终结牌兑现窗口。
- 0 费弱攻击帮助玩家更早触达 5 层连锁，但需要控制基础强度，避免无成本堆连锁过于稳定。2 费强攻击用于体现费用影响伤害和回合节奏，但默认仍只 +1 连锁。
- 初始 2 张基础防御技能牌提供防御值并 +1 行动点，帮助玩家在防御回合继续延展行动牌链条。因为技能牌默认不消耗行动点，这类收益需要通过防御值、抽牌环境和敌人压力共同测试。
- 初始 1 张过牌技能牌抽 2 张牌，用于提高玩家找到行动牌或终结牌的概率，帮助玩家理解过牌对卡组运转的价值。
- 初始终结牌至少需要 3 层连锁才能使用，3 层造成高额单体伤害，5 层连锁及以上造成翻倍伤害。该门槛把终结牌限制从行动点成本转移到连锁层数、抽到时机和是否值得兑现。
- 防御、抽牌、回费、异常状态、群体伤害、保留连锁等特殊终结牌效果应主要通过闯关奖励、构筑转向或流派扩展逐步进入卡池。
- 初始卡组使用保守数值，方便后续通过奖励感到成长明显；具体伤害值、防御值和终结牌基础伤害留到最小原型阶段统一设定。

## 待验证问题

- 是否存在局外货币。
- 普通战斗卡牌包奖励的出现频率、具体稀有度概率和不同节点卡牌包质量。
- 普通遗物、特殊遗物和 Boss 高稀有度卡牌包的价值边界；第一版 MVP 只验证 1 个普通遗物的基础影响。
- 行动点上限提升、当前回合行动点增加和下回合额外行动点的数值边界与 UI 表达。
- 最小原型阶段统一设定初始攻击、防御和终结牌伤害数值。
- 保留手牌、防御、行动点或连锁层数的价值基准。
- 商店刷新的卡牌、遗物、服务比例。
- 治疗、删牌、升级的经济价值基准。
- 是否通过商店、遗物、事件或特殊节点加入跳过 / 少拿卡牌收益。
- 过牌、检索、强制重洗、减费、回费、复制和连锁保留的价值基准。
