# 核心循环

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义玩家从三拍战斗、终结释放、武器卡池成长到构筑成型的主要循环。所有局内系统都应挂接到这条循环上。

## 局内主循环

1. 观察当前状态：生命、无色彩能数量、手牌、玩家拍位、魔物拍位、当前武器卡组和终结槽。
2. 做出战斗选择：将行动牌放入玩家拍位，自动拉出箭头选择魔物拍位或空门本体，决定如何拆招、格挡、闪避和攻击。
3. 结束结算：按拍位目标执行动作对撞；成功造成伤害、有效格挡或成功闪避时生成无色彩能。
4. 使用终结牌：终结槽满足彩能条件后，由玩家手动释放，消耗固定数量、X 数量或全部无色彩能并结算终结效果。
5. 获得战斗奖励：当前实现优先验证武器卡牌三选一；遗物、Boss 简化奖励和色彩碎片附魔作为后续阶段恢复。
6. 面对更高压力：魔物拍位组合、Boss 规则、彩能生产不足、终结牌时机或构筑检验。
7. 通过卡牌、遗物、魔物与事件文本获得黑塔世界观碎片。

## 单局节奏目标

完整版本的单局目标时长为 30-40 分钟。该时长应允许玩家经历武器构筑方向识别、三拍拆招熟练、后续色彩附魔路线建立、彩能引擎逐步成型、终结牌高光和黑塔叙事碎片积累，同时仍保持反复开局的意愿。

第一版 MVP 的目标时长收束为 5-8 分钟，只验证战斗局内循环、主副武器起始牌和最小战后武器卡奖励。MVP 采用 1 个线性章节，战斗顺序为：单目标普通战斗 -> 多目标普通战斗 -> Boss / 精英强度战斗；3 场战斗全部采用固定遭遇，暂不做路线图、商店、休息、普通战斗随机池、精英遗物奖励或完整货币经济。

第一版 MVP 内容策略收束为“左轮剑 + 机械臂”的极小教学池：每种武器提供固定 4 张初始卡牌，玩家在开局界面自由选择主手与副手；随后系统自动将两把武器的 4 张初始卡全部加入初始卡组，合计 8 张。奖励牌优先从这两个武器的当前极小卡池中给出，允许重复获得同名牌，不追求完整武器解锁池。

第一版 MVP 每场战斗开始时玩家生命回满，用于稳定测试单场三拍战斗和无色彩能终结闭环，不测试跨战斗生命损耗。正式版本方向为生命跨战斗保留且不自动恢复；正式版本恢复节点和恢复经济本轮不展开。

第一版 MVP 中生命归零会立即结束本次 MVP Run，回到开局或失败结算；玩家流程不提供当前战斗重试，以保证 Roguelike 体验一致。

第一版 MVP 中 Boss 击败后显示 MVP 通关并提供重开入口，同时加入简化 Boss 奖励展示或领取，用于验证 Boss 遗物、Boss 特有卡牌或色彩核心作为阶段性成长的反馈。

## 奖励与构筑调整

普通战斗后的长期基础奖励方向由旧卡牌包结构替换为双层成长；当前三拍战斗验证切片只接入武器卡牌奖励，色彩碎片附魔延后：

- 色彩碎片：后续阶段每场战斗后随机获得 1 个色彩碎片。玩家可以选择 1 张未附魔行动牌进行附魔，使该行动牌后续生成的彩能带有对应色彩。
- 武器卡牌：每场战斗后提供 3 张卡牌候选，来源于当前主手或副手武器相关卡池。玩家选择 1 张加入卡组。候选应清楚标记所属武器、类型、拍位占用、动作串、彩能生成条件或终结牌彩能消耗方式。

当前 3 场 MVP 中不实现战后附魔奖励，也不实现精英额外遗物。Boss 战斗胜利后直接进入 MVP 通关结算；完整版本中 Boss 奖励应包含 Boss 遗物、Boss 特有卡牌和色彩核心等阶段性构筑转折。

色彩核心用于提升某种色彩对卡牌附魔或终结牌结算的效果，正式版本可作为阶段 Boss 的主要长期局内成长之一。当前三拍切片暂不接入色彩核心，Boss 简化奖励也延后到后续阶段。

## 长期循环

1. 开始新局。
2. 选择主手 / 副手武器，由系统自动组成初始卡组。
3. 探索三拍拆招、无色彩能终结和魔物拍位规律；后续阶段再加入色彩规律。
4. 失败或胜利后获得知识、解锁或挑战目标。
5. 带着新武器选择、三拍编排经验、终结牌使用判断和事件线索再次进入；后续五色阶段再加入附魔思路。

## 设计准则

- 每轮战斗都应提供至少一个有意义选择。
- 奖励选择应引导玩家思考武器卡池、行动牌动作串、成功产彩稳定性和终结牌消耗条件，而不是只选数值最高。
- 色彩附魔是后续构筑方向选择，恢复时不应退化为无脑给最高频行动牌贴最强颜色。
- 普通战斗卡牌奖励默认添加 1 张卡牌；控制卡组厚度的机制留到删牌、事件或特殊奖励中处理。
- 完整版本的路线选择必须能被玩家提前评估一部分风险；第一版 MVP 暂不验证路线选择，只验证线性战斗和奖励循环。
- 随机奖励应在“惊喜”和“可规划”之间取得平衡。
- 精英和 Boss 奖励应明显区别于普通战斗奖励，让玩家愿意承担路线风险。
- 叙事碎片应通过卡牌、遗物、魔物、Boss、事件和结算文案自然出现，不打断战斗节奏。

## 待验证问题

- 每场战斗平均轮数是多少？
- 6 点彩能上限是否足够支撑爽快终结牌，同时避免单轮或整场战斗过长？
- 玩家是否能理解“放入拍位 -> 立即指向目标 -> 对撞 / 空门”的连续操作？
- 后续五色阶段中，5 种色彩的基础效果是否都能形成明确构筑路线？
- 后续五色阶段中，每战 1 次附魔是否过快或过慢？
- 武器卡牌三选一是否需要按主手 / 副手分池展示？
- 后续恢复 Boss 简化奖励时应是固定领取、三选一，还是只展示结算？
