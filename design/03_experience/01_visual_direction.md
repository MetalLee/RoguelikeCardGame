# 视觉方向

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义视觉风格、可读性要求、美术资产优先级和 AI 占位资产生成准则。视觉必须同时服务漫画书感、识别、操作和《剑与黑塔》的失色 / 夺色主题。

## 视觉目标

- 画面风格需要成为与同类 Roguelike 卡牌游戏区分的第一层差异化信号。
- 视觉基调采用“黑白手绘漫画线稿风格”：整体画面应像一本可交互的黑白漫画分镜，使用高留白纸面、手绘线稿、粗细变化明显的墨线、少量排线 / 网点、黑色块面、分格式构图和夸张动作张力。
- 世界基础状态是失色、荒芜和灰白。默认美术资产以黑、白、少量灰阶为主；当前无色彩能、目标指向、终结牌和后续色彩核心是少数允许使用高饱和颜色的机制性例外。
- 默认战斗界面气氛应保持明亮、温暖、可读，避免把 UI 和背景推向恐怖、重黑或厚重装饰方向。
- 战斗演出优先使用漫画分镜语言，例如冲击格、速度线、局部特写、拟声字、跨格打击和命中瞬间的短暂停顿，让玩家产生“正在看一场由自己出牌驱动的漫画战斗”的感觉。
- 卡牌、魔物、状态、色彩和奖励能快速识别。
- 战斗信息层级清楚，不被背景或特效干扰。
- 终结牌释放需要提供明显的动画升级感，让玩家感受到彩能被兑现；后续五色阶段再追加颜色兑现反馈。
- 数值反馈、彩能反馈、目标指向反馈和画面效果需要相互配合，让玩家同时看懂收益并感到爽快。
- 角色和世界观有记忆点。
- 独立游戏制作范围内可持续产出。

## MVP 占位美术最高准则

`design/03_experience/assets/` 下现有图片资源是第一版 MVP 占位美术的最高风格准则。后续所有通过 `gpt-image-2` 或其他 AI 图像工具生成的 MVP 占位美术资源，必须优先对齐这些参考图；若与早期“彩色漫画”“厚重奇幻”“卡通 Q 版”等描述冲突，以本节为准。

核心参考资源：

- [[design/03_experience/assets/视觉参考.jpg|视觉参考]]：总体场景、角色比例、黑白高留白、荒芜地表、黑塔 / 悬浮岩石 / 左轮剑图形语言。
- [[design/03_experience/assets/新背景.png|新背景]]：横版战斗舞台、低地平线、远景烟尘、岩石排线、地表线条和背景低密度细节。运行时应裁掉上方透明区后作为单张 1920×1080 背景接入。
- [[design/03_experience/assets/人物动作动画/人物.png|人物]]、[[design/03_experience/assets/人物动作动画/攻击/连招1.png|连招1]]、[[design/03_experience/assets/人物动作动画/必杀/必杀01.png|必杀01]]：主角身形、动作姿态、服装褶皱、剑招弧线和大面积黑白对比。
- [[design/03_experience/assets/魔物序列帧动画/骷髅1-5.png|骷髅1-5]]、[[design/03_experience/assets/魔物序列帧动画/魔物2动作序列帧动画/魔物2-站立.png|魔物2-站立]]：魔物轮廓、剪影化身体、面部高反差和异常感。
- [[design/03_experience/assets/魔物序列帧动画/BOSS序列帧动画/发射导弹.png|Boss 发射导弹]]、[[design/03_experience/assets/魔物序列帧动画/BOSS序列帧动画/死鲨鱼.png|死鲨鱼]]：Boss 体量、大面积纯黑、少量白色高光和机械 / 异形结合。
- [[design/03_experience/assets/爆炸效果.png|爆炸效果]]：战斗特效应采用黑白烟团、冲击线、白色镂空和漫画化形变，而不是彩色粒子堆叠。

统一风格摘要：

- 色彩：默认黑白灰，白底优先；黑色用于剪影、头发、服装暗部、Boss 体块、UI 墨线和爆炸重心。灰阶只做轻微过渡和高光，不做写实体积渲染。
- 线条：外轮廓用较粗墨线，内部结构用更细的手绘线、短排线和少量断线。线条可以有轻微抖动和手绘不均匀感，但不能脏乱到影响识别。
- 留白：大面积白色是风格的一部分，不需要用纹理填满。背景细节应集中在地平线、烟尘、岩石、黑塔和少数前景物件。
- 明暗：允许高对比纯黑块面，尤其用于魔物、Boss、角色头发、机械结构和冲击格；不要使用复杂彩色光照、油画式渐变或照片级阴影。
- 角色：主角保持瘦长、冷峻、剑客 / 旅者气质，服装以长袍、绑带、靴子、左轮剑和机械臂为识别重点；动作要清楚表达出牌或终结牌的力道。
- 魔物：优先使用清晰剪影和少量怪异符号，不追求恐怖血腥。普通魔物可以简单，精英和 Boss 可以用更大的黑色块面与机械 / 黑塔异化元素拉开层级。
- 背景：横版 16:9 战斗舞台优先，低地平线、荒漠、断裂岩体、烟尘、黑塔和旧文明残片构成失色世界；不要做满屏复杂建筑、彩色天空或重雾暗场。
- 特效：斩击、爆炸、防御、彩能和终结牌特效使用墨线、速度线、烟团、漫画格、白色镂空和少量灰阶。彩能颜色只作为机制强调，不应把整张图染成单一色相。
- UI：UI 底层使用纸面、墨线框、漫画格边界和稳定排版；机制色只用于彩能槽、附魔标记、奖励色彩和关键高亮，不用于大面积背景装饰。

`gpt-image-2` 生成提示词应包含的正向约束：

- `black and white hand-drawn manga line art`
- `high-key white paper background`
- `strong ink outlines with thin interior hatching`
- `large negative space`
- `comic panel composition`
- `transparent background`，仅当生成角色、魔物、图标、VFX 或 UI 零件时使用
- `no rendered color except tiny gameplay accent`，仅当资产确实需要机制色时使用

禁止项：

- 禁止默认生成全彩插画、厚涂、写实 3D、半写实电影概念图、像素风、低多边形、赛璐璐高饱和动画风或厚重阴暗油画风。
- 禁止使用大面积紫蓝渐变、霓虹光污染、照片级镜头光、复杂材质贴图、强环境光、彩色粒子雨或满屏烟雾。
- 禁止把背景画得比角色和魔物更抢眼；禁止让特效遮挡生命、防御、意图、目标选择和手牌。
- 禁止在 AI 生成图中烘焙正式 UI 文本、数值和卡牌规则文本；正式文本必须由 Godot UI 绘制。

验收标准：

- 缩小到游戏内实际显示尺寸后，角色 / 魔物 / 图标轮廓仍然可辨。
- 去掉全部机制色后，资产仍能成立为黑白漫画线稿。
- 与上述参考资源并排查看时，线条密度、留白比例、黑色块面和灰阶使用不突兀。
- 资产可以被透明 PNG 分层接入，不依赖不可裁切的整张插画背景。
- 若资产用于彩能、目标指向、后续附魔或色彩核心，颜色必须只承担机制编码，不改变整体黑白漫画基调。

## 美术资产优先级

1. 卡牌框架、拍位 / 目标指向 / 彩能图标和关键词可读性。
2. 战斗界面和魔物表现。
3. 主角左、左轮剑、机械臂、Boss 和高频奖励。
4. 事件插图、背景和装饰元素。

## 第一版 MVP 关键元素视觉方向

旧版 MVP 已有“剑客主角、魔物、行动牌 / 技能牌 / 终结牌模板、卡牌包、连锁 UI”等资源。其中 `design/03_experience/assets/` 里的黑白线稿资源已经升级为第一版 MVP 占位美术的最高风格参考；旧卡牌类型、卡包和连锁 UI 语义只作为历史实现参考，需要按新设计迁移：

- 主角：从泛用剑客明确为「左」。形象应体现剑灵化人、孤独旅者、左轮剑携带者和机械臂使用者。当前 active 战斗站姿使用从 `design/03_experience/assets/人物动作动画/人物.png` 切分出的持左轮剑造型，资源路径为 `game/assets/art/characters/zu_revolver_sword.png`；同源无武器造型保留为 `game/assets/art/characters/zu_unarmed.png`。
- 左轮剑：剑与枪的结合，是主角身份和茵沉睡线索的重要视觉锚点。
- 机械臂：高科技义肢，应与失色世界形成科技感反差，并承担护盾 / 控制视觉。
- 卡牌整体：保留竖版卡牌比例、安全边距和漫画分镜插画。
- 行动牌：不再只等于攻击牌，也承载攻击、格挡、闪避、控制、多段动作和彩能生成条件。当前 active 行动牌模板为 600×802 黑白漫画卡框，卡框需要能显示所属武器、动作串、拍位占用和后续附魔色彩；费用、卡名、机制摘要和规则文本由 Godot UI 按模板安全区动态绘制。
- 终结牌：显示彩能需求或消耗规则，而不是固定连锁阈值。所有终结牌都应拥有独立美术资源，表达目标范围、核心收益和色彩兑现感。
- 色彩碎片：后续五色阶段需要红、黄、蓝、绿、紫 5 种清楚可辨的奖励图标，但图标主体仍应是黑白墨线造型，颜色只作为机制填色或边缘高亮。
- 色彩核心：比碎片更高阶的 Boss 奖励，应有更强视觉权重；允许更醒目的机制色，但必须保留黑白墨线主体。
- 遗物：遵循漫画风格，轮廓清楚，并可隐晦表达剧情。
- 魔物：统一到黑塔异常、勇者联盟腐败、失色生态或旧文明残留，不再停留在抽象魔物占位。

## 战斗场景设计标准

后续战斗界面实现与视觉迭代应优先遵循以下方向：

- 使用明亮纸张、清晰墨线和轻量网点组织画面，不以恐怖符号或大面积黑色装饰作为默认战斗 UI 气质。
- 失色世界的背景应以黑白灰线稿为主；交互重点、无色彩能、目标指向、后续色彩碎片 / 附魔和终结牌可以使用鲜明颜色，但颜色面积应克制，避免破坏“失色世界”基调。
- MVP 运行时战斗背景以白色纸面为主背景色，并由 `asset.background.mvp_battle` 单张 1920×1080 透明 PNG 承载远景烟尘、岩体、黑塔线索和地表线条。主角站立脚底需要对齐背景地面线，避免悬空。
- 主角和魔物应站在同一水平战斗线上，中心舞台保留足够垂直空间，确保更高的精英和 Boss 可以替换进入而不破坏 HUD。
- 顶部 HUD 不使用主角或魔物头像，避免占用舞台空间；玩家和魔物的生命、防御、状态与意图通过条形 UI、图标和文字表达。
- 手牌保持竖版卡牌比例并在底部以紧凑扇形叠放。
- 彩能 UI 使用 6 个基础槽位表达当前无色彩能。后续五色阶段再通过槽位颜色展示无色 / 红 / 黄 / 蓝 / 绿 / 紫。

## 运行时组合规则

Godot 表现层应通过 `game/data/presentation/` 组合视觉资源。卡牌在运行时由类型模板、武器标记、卡面插画、规则文本、拍位 / 目标状态、后续附魔色彩和状态高亮叠加生成；魔物由遭遇实例和 enemy view 生成；奖励卡牌、遗物以及后续色彩碎片 / 色彩核心由对应 view 显示。

新增美术资源只有进入 asset manifest 并被 view 引用后，才视为已接入游戏。

通过 `gpt-image-2` 生成的 MVP 占位资源进入工程前，需要先按本文件的 MVP 占位美术最高准则自检，并在资源归档记录或 asset manifest 备注中标明其参考了 `design/03_experience/assets/` 风格锁定。未通过风格自检的图像只能保留在临时目录，不应进入正式表现 view。

动画必须由规则层日志事件触发，并保持核心 HUD、魔物意图、手牌和目标选择区域稳定；不能为了演出移动或遮挡玩家下一步决策所需的信息。

## 设计准则

- 漫画书感不能压低可读性；卡牌文本、魔物意图、拍位目标、状态图标、伤害数字和彩能必须保持清楚。
- 色彩应是机制信息，不只是装饰。当前先保证无色彩能与目标指向清楚；后续五色阶段红、黄、蓝、绿、紫必须在卡牌、彩能槽、奖励和结算中保持统一视觉编码。
- 黑白线稿是世界默认视觉语言；颜色是被夺回的机制信息，不是常规装饰。
- 手绘漫画风格应服务动作张力和终结牌演出，例如夸张命中姿态、速度线、分格式冲击、局部特写、拟声字或高对比轮廓。
- UI 可以使用漫画格、页边、墨线框和纸张纹理作为结构语言，但核心状态信息必须保持稳定位置和稳定尺寸。
- 特效应强调结果和节奏，不遮挡关键数值。
- 终结牌释放应拥有逐级增强的视觉反馈；后续五色阶段再追加色彩兑现反馈。
- 主角出牌动作应由卡牌效果和所属武器驱动，而不是只由卡牌类型驱动。
- 巨额伤害和夸张特效可以作为核心爽感，但必须保持敌我生命、伤害数字、意图和后续操作可读。

## 待验证问题

- 失色世界背景与高饱和色彩机制如何平衡，不让画面脏或过花。
- 左、茵、左轮剑和机械臂的稳定视觉识别。
- 5 种色彩在小尺寸 UI 中的可辨识度。
- 是否采用 2D、伪 3D 或混合表现。
