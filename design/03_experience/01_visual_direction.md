# 视觉方向

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义视觉风格、可读性要求和美术资产优先级。视觉必须同时服务漫画书感、识别、操作和《剑与黑塔》的失色 / 夺色主题。

## 视觉目标

- 画面风格需要成为与同类 Roguelike 卡牌游戏区分的第一层差异化信号。
- 视觉基调采用“手绘漫画书风格”：整体画面应像一本可交互漫画，使用手绘线稿、网点阴影、纸张纹理、粗细变化明显的墨线、分格式构图和夸张动作张力。
- 世界基础状态是失色、荒芜和灰白，但玩家战斗中的彩能、附魔、终结牌和色彩核心应形成鲜明色彩回归。
- 默认战斗界面气氛应保持明亮、温暖、可读，避免把 UI 和背景推向纯暗黑哥特、恐怖或重黑装饰方向。
- 战斗演出优先使用漫画分镜语言，例如冲击格、速度线、局部特写、拟声字、跨格打击和命中瞬间的短暂停顿，让玩家产生“正在看一场由自己出牌驱动的漫画战斗”的感觉。
- 卡牌、敌人、状态、色彩和奖励能快速识别。
- 战斗信息层级清楚，不被背景或特效干扰。
- 终结牌释放需要提供明显的动画升级感，让玩家感受到彩能和色彩被兑现。
- 数值反馈、彩能反馈、色彩反馈和画面效果需要相互配合，让玩家同时看懂收益并感到爽快。
- 角色和世界观有记忆点。
- 独立游戏制作范围内可持续产出。

## 美术资产优先级

1. 卡牌框架、彩能 / 色彩图标和关键词可读性。
2. 战斗界面和敌人表现。
3. 主角左、左轮剑、机械臂、Boss 和高频奖励。
4. 事件插图、背景和装饰元素。

## 第一版 MVP 关键元素视觉方向

旧版 MVP 已有“剑客主角、恶魔敌人、行动牌 / 技能牌 / 终结牌模板、卡牌包、连锁 UI”等资源。这些资源可作为手绘漫画书风格参考，但需要按新设计迁移：

- 主角：从泛用剑客明确为「左」。形象应体现剑灵化人、孤独旅者、左轮剑携带者和机械臂使用者。
- 左轮剑：剑与枪的结合，是主角身份和茵沉睡线索的重要视觉锚点。
- 机械臂：高科技义肢，应与失色世界形成科技感反差，并承担护盾 / 控制视觉。
- 卡牌整体：保留竖版卡牌比例、安全边距和漫画分镜插画。
- 行动牌：不再只等于攻击牌，也承载防御、控制、过牌、回费、减费和彩能生成。卡框需要能显示行动点费用、所属武器和附魔色彩。
- 终结牌：显示彩能需求或消耗规则，而不是固定连锁阈值。所有终结牌都应拥有独立美术资源，表达目标范围、核心收益和色彩兑现感。
- 色彩碎片：需要红、黄、蓝、绿、紫 5 种清楚可辨的奖励图标。
- 色彩核心：比碎片更高阶的 Boss 奖励，应有更强视觉权重。
- 遗物：遵循漫画风格，轮廓清楚，并可隐晦表达剧情。
- 敌人：统一到黑塔异常、勇者联盟腐败、失色生态或旧文明残留，不再停留在抽象恶魔占位。

## 战斗场景设计标准

后续战斗界面实现与视觉迭代应优先遵循以下方向：

- 使用明亮纸张、清晰墨线和轻量网点组织画面，不以暗黑哥特、恐怖符号或大面积黑色装饰作为默认战斗 UI 气质。
- 失色世界的背景可偏灰白，但交互重点、彩能、色彩碎片、附魔和终结牌应使用鲜明颜色。
- 主角和敌人应站在同一水平战斗线上，中心舞台保留足够垂直空间，确保更高的精英和 Boss 可以替换进入而不破坏 HUD。
- 顶部 HUD 不使用主角或敌人头像，避免占用舞台空间；玩家和敌人的生命、防御、状态与意图通过条形 UI、图标和文字表达。
- 手牌保持竖版卡牌比例并在底部以紧凑扇形叠放。
- 彩能 UI 使用 6 个基础槽位表达当前彩能，并通过槽位颜色展示无色 / 红 / 黄 / 蓝 / 绿 / 紫。

## 运行时组合规则

Godot 表现层应通过 `game/data/presentation/` 组合视觉资源。卡牌在运行时由类型模板、武器标记、卡面插画、规则文本、附魔色彩和状态高亮叠加生成；敌人由遭遇实例和 enemy view 生成；色彩碎片、色彩核心、奖励卡牌和遗物由对应 view 显示。

新增美术资源只有进入 asset manifest 并被 view 引用后，才视为已接入游戏。

动画必须由规则层日志事件触发，并保持核心 HUD、敌人意图、手牌和目标选择区域稳定；不能为了演出移动或遮挡玩家下一步决策所需的信息。

## 设计准则

- 漫画书感不能压低可读性；卡牌文本、敌人意图、状态图标、伤害数字、彩能和色彩构成必须保持清楚。
- 色彩应是机制信息，不只是装饰。红、黄、蓝、绿、紫必须在卡牌、彩能槽、奖励和结算中保持统一视觉编码。
- 手绘漫画风格应服务动作张力和终结牌演出，例如夸张命中姿态、速度线、分格式冲击、局部特写、拟声字或高对比轮廓。
- UI 可以使用漫画格、页边、墨线框和纸张纹理作为结构语言，但核心状态信息必须保持稳定位置和稳定尺寸。
- 特效应强调结果和节奏，不遮挡关键数值。
- 终结牌释放和色彩兑现应拥有逐级增强的视觉反馈。
- 主角出牌动作应由卡牌效果和所属武器驱动，而不是只由卡牌类型驱动。
- 巨额伤害和夸张特效可以作为核心爽感，但必须保持敌我生命、伤害数字、意图和后续操作可读。

## 待验证问题

- 失色世界背景与高饱和色彩机制如何平衡，不让画面脏或过花。
- 左、茵、左轮剑和机械臂的稳定视觉识别。
- 5 种色彩在小尺寸 UI 中的可辨识度。
- 是否采用 2D、伪 3D 或混合表现。
