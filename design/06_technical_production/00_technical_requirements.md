# 技术需求

状态：草案  
上级索引：[[design/README|Design Knowledge Base]]

## 目的

定义第一版 MVP 和后续开发的技术边界、架构原则、工具需求和验证方式。

## 当前实现状态提示

截至 2026-06-13，项目已有 Godot 4.6.x .NET / C# 旧版 MVP 原型，实现了旧“连锁层数、技能牌、卡牌包奖励、3 / 5 / 8 阈值”方向。该实现可作为工程基础和历史原型，但已与新的《剑与黑塔》彩能 / 色彩 / 武器卡池设计不一致，后续开发需要重构规则层、数据、UI 和测试。

## 技术原则

- Godot 工程根目录固定为 `game/`。
- 玩法规则层与 Godot 表现层分离。
- C# 规则逻辑不要依赖 Godot 节点树。
- 卡牌、武器、敌人、遗物、遭遇、奖励、色彩和本地化优先数据驱动，不要把内容数值写死在场景里。
- 表现层只读取规则层结果和结构化日志，不自行结算伤害、彩能、色彩、费用、防御或胜负。
- 每次新增或修改代码、数据时，同步补充可运行校验、测试或最小验证方式。

## 第一版 MVP 必须支持

- 主角「左」。
- 起始武器池：左轮剑、机械臂。
- 起始主副武器选择：玩家可自由选择左轮剑或机械臂作为主手，另一个作为副手。
- 起始武器选牌：主手武器 8 选 6，副手武器 8 选 4，组成 10 张初始卡组。
- 行动牌和终结牌两类卡牌；不再保留技能牌类型。
- 基础行动点 3。
- 彩能基础上限 6，跨回合不保留。
- 行动牌生成彩能，并可根据附魔生成带颜色彩能。
- 5 种色彩：红、黄、蓝、绿、紫。
- 终结牌按自身数据消耗固定数量、X 数量或全部彩能，并根据被消耗彩能颜色结算附加效果。
- 普通战斗后奖励：随机色彩碎片 + 武器卡牌三选一。
- 精英战斗后额外普通遗物。
- Boss 击败后通关结算 + 简化 Boss 奖励。
- 每场战斗开始时玩家生命回满；生命归零立即结束本次 MVP Run。
- 6 场固定遭遇，敌人语境统一到黑塔世界观。

## 规则层需求

规则层应至少包含：

- `WeaponDefinition`：武器 ID、初始卡池、奖励卡池、武器标签。
- `CardDefinition`：所属武器、卡牌类型、费用、目标、彩能生成 / 消耗、色彩互动、效果 DSL。
- `ColorDefinition` 或等价枚举：红、黄、蓝、绿、紫与无色。
- `ColorShard` / 附魔状态：记录行动牌是否已附魔及其颜色。
- `CombatState`：生命、防御、行动点、彩能槽、牌区、敌人、日志。
- `RunState`：当前武器、主卡组、附魔状态、遗物、当前节点、Boss 奖励状态。
- `CombatLogEvent`：记录出牌、目标、行动点变化、彩能变化、色彩结算、防御变化、抽弃牌、敌人意图和触发效果。

## 表现层需求

- 支持起始主副武器选择界面和起始武器选牌界面。
- 支持 6 格基础彩能槽，并显示每点彩能颜色。
- 支持行动牌附魔标记。
- 支持终结牌预览：消耗彩能数量、颜色构成、预计伤害或资源收益、结算后剩余彩能。
- 支持色彩碎片奖励和附魔界面。
- 支持武器卡牌三选一奖励。
- 支持 Boss 简化奖励结算。
- 动画、音效和屏幕反馈应由结算事件驱动，避免表现层与规则层各算一套结果。

## 数据与工具需求

- JSON + JSON Schema 仍作为首版内容数据格式。
- 数据校验必须检查内容 ID 唯一、引用存在、武器卡池数量、卡牌类型、彩能消耗、色彩引用、奖励候选和本地化文本。
- 战斗沙盒需要支持：选择固定遭遇、选择起始武器卡牌、设置随机种子、设置初始生命 / 行动点 / 彩能、指定色彩碎片、加入指定遗物。
- 试玩指标导出需要记录彩能、色彩、附魔和终结牌消耗。

## 项目目录

根据 [[design/08_governance/2026-06-04_keep_godot_project_in_same_repo_game_subdir|Godot 工程与知识库同仓库并放入 game 子目录]]，第一版 MVP 阶段 Godot 工程与知识库保留在同一个 Git 仓库中，但 Godot 工程根目录固定为 `game/`。知识库根目录继续维护 `design/`、`inspiration/`、`insight/` 等文档目录，游戏实现相关文件放在 `game/` 下。

## 验证要求

- 数据校验通过。
- 规则层单元测试覆盖起始武器选牌、行动牌生成彩能、附魔、终结牌消耗彩能、五色附加效果、战后奖励、精英遗物、Boss 奖励和失败结算。
- Godot headless 或可用启动验证通过。
- 若当前实现仍未迁移，应在验证结果中明确“旧 MVP 通过，新设计未实现”。
